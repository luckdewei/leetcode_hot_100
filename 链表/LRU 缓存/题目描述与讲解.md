# LRU 缓存（LRU Cache）

- 题目链接：https://leetcode.cn/problems/lru-cache/
- 难度：中等
- 标签：设计、哈希表、链表、双向链表

## 题目描述

设计 LRU（最近最少使用）缓存：支持 `get` 和 `put`，均为 O(1) 平均时间。

- `get(key)`：key 存在返回 value，否则 -1
- `put(key, value)`：更新或插入；超出容量时淘汰最久未使用的项

### 示例

```
输入：["LRUCache","put","put","get","put","get","put","get","get","get"]
     [[2],[1,1],[2,2],[1],[3,3],[2],[4,4],[1],[3],[4]]
输出：[null,null,null,1,null,-1,null,-1,3,4]
```

### 约束

- `1 <= capacity <= 3000`
- `0 <= key <= 10^4`
- `0 <= value <= 10^5`
- 最多 `2 * 10^5` 次调用

---

## 解法一：OrderedDict（推荐）

### 思路

Python `OrderedDict`：`move_to_end(key)` 标记最近使用；超容量时 `popitem(last=False)` 淘汰最久未用。

### 代码

```python
from collections import OrderedDict


class LRUCache:
    def __init__(self, capacity: int):
        self.cap = capacity
        self.cache = OrderedDict()

    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1
        self.cache.move_to_end(key)
        return self.cache[key]

    def put(self, key: int, value: int) -> None:
        if key in self.cache:
            self.cache.move_to_end(key)
        self.cache[key] = value
        if len(self.cache) > self.cap:
            self.cache.popitem(last=False)
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| get/put | 均摊 O(1) |
| 空间 | O(capacity) |

### 讲解

面试中 Python 可用 OrderedDict 快速实现。Java 需手写双向链表 + HashMap。

## 解法二：哈希表 + 双向链表

### 思路

HashMap 存 key→节点；双向链表按使用时间排序，头最近、尾最久。get/put 时 splice 节点到头部。

### 代码

```python
class Node:
    def __init__(self, key=0, val=0):
        self.key = key
        self.val = val
        self.prev = None
        self.next = None


class LRUCache:
    def __init__(self, capacity: int):
        self.cap = capacity
        self.cache = {}
        self.head = Node()
        self.tail = Node()
        self.head.next = self.tail
        self.tail.prev = self.head

    def _remove(self, node):
        node.prev.next = node.next
        node.next.prev = node.prev

    def _add_to_head(self, node):
        node.next = self.head.next
        node.prev = self.head
        self.head.next.prev = node
        self.head.next = node

    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1
        node = self.cache[key]
        self._remove(node)
        self._add_to_head(node)
        return node.val

    def put(self, key: int, value: int) -> None:
        if key in self.cache:
            self._remove(self.cache[key])
        node = Node(key, value)
        self.cache[key] = node
        self._add_to_head(node)
        if len(self.cache) > self.cap:
            lru = self.tail.prev
            self._remove(lru)
            del self.cache[lru.key]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| get/put | O(1) |
| 空间 | O(capacity) |

### 讲解

标准 LRU 实现，非 Python 语言面试必写。哑头哑尾简化边界。

## 解法三：仅用字典（不满足 LRU）

### 思路

普通 dict，超容量时删第一个插入的 key（Py3.7+ 保序但非访问序）。

### 代码

```python
class LRUCache:
    def __init__(self, capacity: int):
        self.cap = capacity
        self.cache = {}

    def get(self, key: int) -> int:
        return self.cache.get(key, -1)

    def put(self, key: int, value: int) -> None:
        if len(self.cache) >= self.cap and key not in self.cache:
            self.cache.pop(next(iter(self.cache)))
        self.cache[key] = value
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| get/put | O(1) |
| 空间 | O(capacity) |

### 讲解

get 不更新顺序，不满足 LRU 语义，仅作错误示范。

## 解法对比

| 解法 | get/put | 实现难度 | 适用场景 |
|------|---------|----------|----------|
| OrderedDict | O(1) | 低 | Python 面试 |
| 双向链表 | O(1) | 高 | 通用语言 |
| 普通 dict | O(1) | 低 | 不满足题意 |

---

## 常见错误

1. **get 后不更新顺序**：应 move_to_end。
2. **put 已存在 key 未更新顺序**：同样要移到最新。
3. **淘汰时删错端**：应删 `last=False`（最久未用）。
