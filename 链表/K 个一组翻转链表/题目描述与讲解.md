# K 个一组翻转链表（Reverse Nodes in k-Group）

- 题目链接：https://leetcode.cn/problems/reverse-nodes-in-k-group/
- 难度：困难
- 标签：递归、链表

## 题目描述

给你链表的头节点 `head`，每 k 个节点一组进行翻转，返回修改后的链表。k 为正整数且 `k <= n`。不足 k 个的剩余节点保持原顺序。

### 示例

```
输入：head = [1,2,3,4,5], k = 2
输出：[2,1,4,3,5]

输入：head = [1,2,3,4,5], k = 3
输出：[3,2,1,4,5]
```

### 约束

- 链表节点数 `n` 范围 `[0, 5000]`
- `0 <= Node.val <= 1000`
- `1 <= k <= n`

---

## 解法一：迭代分组反转（推荐）

### 思路

用哑节点，每次检查是否有 k 个节点，有则反转这一段并连接前后。`_get_kth` 找第 k 个节点作为组尾。

### 代码

```python
from typing import Optional


class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


class Solution:
    def reverseKGroup(self, head: Optional[ListNode], k: int) -> Optional[ListNode]:
        dummy = ListNode(0, head)
        group_prev = dummy
        while True:
            kth = self._get_kth(group_prev, k)
            if not kth:
                break
            group_next = kth.next
            prev, cur = kth.next, group_prev.next
            while cur is not group_next:
                nxt = cur.next
                cur.next = prev
                prev = cur
                cur = nxt
            tmp = group_prev.next
            group_prev.next = kth
            group_prev = tmp
        return dummy.next

    def _get_kth(self, cur: ListNode, k: int) -> Optional[ListNode]:
        while cur and k > 0:
            cur = cur.next
            k -= 1
        return cur
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(1) |

### 讲解

每组反转后，`group_prev` 移到下一组前驱。指针操作较多，建议画图模拟。

## 解法二：递归

### 思路

先检查前 k 个是否存在，存在则反转并递归处理后续，将反转后的尾节点连到递归结果。

### 代码

```python
from typing import Optional


class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


class Solution:
    def reverseKGroup(self, head: Optional[ListNode], k: int) -> Optional[ListNode]:
        cur = head
        for _ in range(k):
            if not cur:
                return head
            cur = cur.next
        prev, cur = None, head
        for _ in range(k):
            nxt = cur.next
            cur.next = prev
            prev = cur
            cur = nxt
        head.next = self.reverseKGroup(cur, k)
        return prev
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(n/k) 递归栈 |

### 讲解

代码更短，但递归栈 O(n/k)。面试迭代写法更常考。

## 解法三：先收集再重建（理解用）

### 思路

转数组，每 k 个反转，再建链表。

### 代码

```python
from typing import Optional, List


class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


class Solution:
    def reverseKGroup(self, head: Optional[ListNode], k: int) -> Optional[ListNode]:
        vals: List[int] = []
        while head:
            vals.append(head.val)
            head = head.next
        i = 0
        while i + k <= len(vals):
            vals[i:i + k] = reversed(vals[i:i + k])
            i += k
        dummy = ListNode(0)
        cur = dummy
        for v in vals:
            cur.next = ListNode(v)
            cur = cur.next
        return dummy.next
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(n) |

### 讲解

不满足 O(1) 空间，但有助于理解分组逻辑。

## 常见错误

1. **不足 k 个仍反转**：应先检查 kth 存在。
2. **连接前后组断链**：反转后 `group_prev.next` 和 `group_prev` 更新易错。
3. **与「两两交换」混淆**：24 题 k=2 的特例。
