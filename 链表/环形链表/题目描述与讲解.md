# 环形链表（Linked List Cycle）

- 题目链接：https://leetcode.cn/problems/linked-list-cycle/
- 难度：简单
- 标签：哈希表、链表、双指针

## 题目描述

给定一个链表，判断链表中是否有环。如果链表中有某个节点，可以通过连续跟踪 `next` 指针再次到达，则存在环。

### 示例

```
输入：head = [3,2,0,-4], pos = 1
输出：true
解释：尾节点连到下标 1 的节点。

输入：head = [1,2], pos = 0
输出：true

输入：head = [1], pos = -1
输出：false
```

### 约束

- 节点数 `[0, 10^4]`
- `-10^5 <= Node.val <= 10^5`
- `pos` 为 -1 或有效下标（`-1` 表示无环）

---

## 解法一：Floyd 快慢指针（推荐）

### 思路

慢指针每次一步，快指针每次两步。若有环，两指针必相遇；若无环，快指针先到 NULL。

### 代码

```python
from typing import Optional


class ListNode:
    def __init__(self, x):
        self.val = x
        self.next = None


class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        slow = fast = head
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
            if slow is fast:
                return True
        return False
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(1) |

### 讲解

Floyd 判圈算法，O(1) 空间最优解。相遇只说明有环，不给出环入口。

---

## 解法二：哈希集合

### 思路

遍历链表，节点入 set，若已存在则存在环。

### 代码

```python
from typing import Optional, Set


class ListNode:
    def __init__(self, x):
        self.val = x
        self.next = None


class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        seen: Set[ListNode] = set()
        while head:
            if head in seen:
                return True
            seen.add(head)
            head = head.next
        return False
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(n) |

### 讲解

直观但 O(n) 空间，不满足 follow-up O(1) 要求。

---

## 解法三：修改节点标记（不推荐）

### 思路

遍历时给节点打标记（如改 val 或附加 visited 属性），再次访问则存在环。

### 代码

```python
from typing import Optional


class ListNode:
    def __init__(self, x):
        self.val = x
        self.next = None


class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        while head:
            if getattr(head, "visited", False):
                return True
            head.visited = True
            head = head.next
        return False
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(1) |

### 讲解

修改了输入结构，面试中不推荐，仅作思路拓展。

---

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 快慢指针 | O(n) | O(1) | 面试首选 |
| 哈希集合 | O(n) | O(n) | 直观写法 |
| 标记法 | O(n) | O(1) | 不推荐 |

## 常见错误

1. **快指针判空**：需 `while fast and fast.next`。
2. **用 `==` 比较值而非节点**：应 `slow is fast` 比较引用。
3. **与「环形链表 II」混淆**：142 题还要求找环入口。
