# 搜索旋转排序数组（Search in Rotated Sorted Array）

- 题目链接：https://leetcode.cn/problems/search-in-rotated-sorted-array/
- 难度：中等
- 标签：数组、二分查找

## 题目描述

整数数组 `nums` 按升序排列，数组中的值**互不相同**。数组在某个下标处进行了旋转。

给你旋转后的数组 `nums` 和一个整数 `target`，若存在则返回下标，否则返回 `-1`。要求 **O(log n)** 时间。

### 示例

```
输入：nums = [4,5,6,7,0,1,2], target = 0
输出：4

输入：nums = [4,5,6,7,0,1,2], target = 3
输出：-1

输入：nums = [1], target = 0
输出：-1
```

### 约束

- `1 <= nums.length <= 5000`
- `-10^4 <= nums[i], target <= 10^4`
- `nums` 各值互不相同，为旋转后的升序数组

---

## 解法一：二分判断有序半段（推荐）

### 思路

取中点 `mid`，左半或右半必有一段有序。判断 `target` 是否在有序段内，决定收缩方向。

### 代码

```python
from typing import List


class Solution:
    def search(self, nums: List[int], target: int) -> int:
        left, right = 0, len(nums) - 1
        while left <= right:
            mid = (left + right) // 2
            if nums[mid] == target:
                return mid
            if nums[left] <= nums[mid]:
                if nums[left] <= target < nums[mid]:
                    right = mid - 1
                else:
                    left = mid + 1
            else:
                if nums[mid] < target <= nums[right]:
                    left = mid + 1
                else:
                    right = mid - 1
        return -1
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(log n) |
| 空间 | O(1) |

### 讲解

`nums[left] <= nums[mid]` 用 `<=` 处理 `left == mid`。先定有序半段，再用区间边界判断 `target` 归属。

## 解法二：先找旋转点再二分

### 思路

先二分找最小值下标（旋转点），再判断 `target` 在左段还是右段，分别标准二分。

### 代码

```python
from typing import List


class Solution:
    def search(self, nums: List[int], target: int) -> int:
        n = len(nums)
        lo, hi = 0, n - 1
        while lo < hi:
            mid = (lo + hi) // 2
            if nums[mid] > nums[hi]:
                lo = mid + 1
            else:
                hi = mid
        pivot = lo

        def bs(l, r):
            while l <= r:
                mid = (l + r) // 2
                if nums[mid] == target:
                    return mid
                if nums[mid] < target:
                    l = mid + 1
                else:
                    r = mid - 1
            return -1

        if nums[pivot] <= target <= nums[-1]:
            return bs(pivot, n - 1)
        return bs(0, pivot - 1)
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(log n) |
| 空间 | O(1) |

### 讲解

两步二分，结构清晰，适合理解旋转数组。无重复元素时旋转点唯一。

## 解法三：线性扫描（理解用）

### 思路

遍历数组找 `target`，不满足 O(log n) 要求。

### 代码

```python
from typing import List


class Solution:
    def search(self, nums: List[int], target: int) -> int:
        for i, x in enumerate(nums):
            if x == target:
                return i
        return -1
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(1) |

### 讲解

仅用于验证，面试不可提交。

## 常见错误

1. **有序判断用 `<` 而非 `<=`**：`left == mid` 时左段仍有序。
2. **边界含等号遗漏**：`target` 与 `nums[left]`、`nums[right]` 比较要含等号。
3. **与 II 题混淆**：81 题有重复元素需额外处理。
