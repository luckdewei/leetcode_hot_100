# 寻找旋转排序数组中的最小值（Find Minimum in Rotated Sorted Array）

- 题目链接：https://leetcode.cn/problems/find-minimum-in-rotated-sorted-array/
- 难度：中等
- 标签：数组、二分查找

## 题目描述

已知一个长度为 `n` 的数组，预先按照升序排列，经由 `1` 到 `n` 次旋转得到。数组中**无重复**元素。

旋转：最后一个元素移到开头，如 `[0,1,2,4,5,6,7]` → `[4,5,6,7,0,1,2]`。

给你旋转后的数组 `nums`，返回其中的**最小元素**。要求 O(log n)。

### 示例

```
输入：nums = [3,4,5,1,2]
输出：1

输入：nums = [4,5,6,7,0,1,2]
输出：0

输入：nums = [11,13,15,17]
输出：11
```

### 约束

- `n == nums.length`
- `1 <= n <= 5000`
- `-5000 <= nums[i] <= 5000`
- `nums` 为旋转升序且无重复

---

## 解法一：线性扫描

### 代码

```python
from typing import List


class Solution:
    def findMin(self, nums: List[int]) -> int:
        return min(nums)
```

### 复杂度

O(n)，不符合要求。

---

## 解法二：二分找旋转点（推荐）

### 思路

最小值是「旋转点」——唯一使 `nums[i] < nums[i-1]` 的位置（未旋转则最小值在 index 0）。

比较 `nums[mid]` 与 `nums[right]`：
- 若 `nums[mid] > nums[right]`：最小值在 mid **右侧** → `left = mid + 1`
- 否则：最小值在 mid **左侧或就是 mid** → `right = mid`

循环 `left < right`，结束时 `nums[left]` 为最小值。

### 代码

```python
from typing import List


class Solution:
    def findMin(self, nums: List[int]) -> int:
        left, right = 0, len(nums) - 1
        while left < right:
            mid = (left + right) // 2
            if nums[mid] > nums[right]:
                left = mid + 1
            else:
                right = mid
        return nums[left]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(log n) |
| 空间 | O(1) |

### 讲解

**为何与 right 比而不是 left？**  
`nums[mid]` 与 `nums[right]` 比较能明确最小值是否在中点右侧：若 mid 值大于 right，说明 mid 在较大那一段，最小值必在 mid 右边。

**未旋转数组** `[11,13,15,17]`：`nums[mid] <= nums[right]` 始终成立，最终 left=0。

## 解法三：找峰值再取后继

找局部最大值下标 k，最小值在 `(k+1) % n`。思路等价，实现更绕。

### 代码

```python
from typing import List


class Solution:
    def findMin(self, nums: List[int]) -> int:
        n = len(nums)
        if nums[0] < nums[-1]:
            return nums[0]
        lo, hi = 0, n - 1
        while lo < hi:
            mid = (lo + hi) // 2
            if nums[mid] > nums[mid + 1]:
                return nums[mid + 1]
            if nums[mid] < nums[lo]:
                hi = mid
            else:
                lo = mid + 1
        return nums[lo]
```

## 解法对比

| 解法 | 时间 | 说明 |
|------|------|------|
| min() / 线性 | O(n) | 简单 |
| 二分 | O(log n) | 标准 |

## 常见错误

1. **用 `right = mid - 1`**：可能跳过最小值，应 `right = mid`。
2. **循环写成 `left <= right`**：与 `left < right` 返回值不同，需统一模板。
3. **153 与 154**：154 允许重复，需额外处理 `nums[mid]==nums[right]`。

## 举一反三

- **搜索旋转排序数组**：先找最小值再二分，或一次二分
- **153 → 154**：重复元素破坏单调性
