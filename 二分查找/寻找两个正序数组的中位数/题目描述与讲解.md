# 寻找两个正序数组的中位数（Median of Two Sorted Arrays）

- 题目链接：https://leetcode.cn/problems/median-of-two-sorted-arrays/
- 难度：困难
- 标签：数组、二分查找、分治

## 题目描述

给定两个大小分别为 `m` 和 `n` 的正序（从小到大）数组 `nums1` 和 `nums2`。请你找出并返回这两个正序数组的**中位数**。

算法的时间复杂度应为 O(log (m+n))。

### 示例

```
输入：nums1 = [1,3], nums2 = [2]
输出：2.00000
解释：合并 = [1,2,3]，中位数 2

输入：nums1 = [1,2], nums2 = [3,4]
输出：2.50000
解释：合并 = [1,2,3,4]，中位数 (2+3)/2 = 2.5
```

### 约束

- `nums1.length == m`, `nums2.length == n`
- `0 <= m <= 1000`, `0 <= n <= 1000`, `1 <= m + n <= 2000`
- `-10^6 <= nums1[i], nums2[i] <= 10^6`

---

## 解法一：合并后取中位数

### 思路

双指针合并两个有序数组，取中间位置。

### 代码

```python
from typing import List


class Solution:
    def findMedianSortedArrays(self, nums1: List[int], nums2: List[int]) -> float:
        merged = []
        i = j = 0
        while i < len(nums1) and j < len(nums2):
            if nums1[i] <= nums2[j]:
                merged.append(nums1[i])
                i += 1
            else:
                merged.append(nums2[j])
                j += 1
        merged.extend(nums1[i:])
        merged.extend(nums2[j:])
        n = len(merged)
        mid = n // 2
        if n % 2 == 1:
            return float(merged[mid])
        return (merged[mid - 1] + merged[mid]) / 2.0
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(m + n) |
| 空间 | O(m + n) |

不符合 O(log(m+n)) 要求，但最易理解。

## 解法二：双指针找第 k 小（分治）

### 思路

中位数即第 `(m+n+1)//2` 小与第 `(m+n+2)//2` 小的平均（偶数长度）。

递归比较两数组第 k/2 位置的元素，排除较小数组的前 k/2 段。

时间 O(log(m+n))，实现较繁琐。

### 代码

```python
from typing import List


class Solution:
    def findMedianSortedArrays(self, nums1: List[int], nums2: List[int]) -> float:
        def find_kth(k: int) -> int:
            i = j = 0
            while True:
                if i == len(nums1):
                    return nums2[j + k - 1]
                if j == len(nums2):
                    return nums1[i + k - 1]
                if k == 1:
                    return min(nums1[i], nums2[j])

                half = k // 2
                i_idx = min(i + half - 1, len(nums1) - 1)
                j_idx = min(j + half - 1, len(nums2) - 1)
                if nums1[i_idx] <= nums2[j_idx]:
                    k -= i_idx - i + 1
                    i = i_idx + 1
                else:
                    k -= j_idx - j + 1
                    j = j_idx + 1

        total = len(nums1) + len(nums2)
        if total % 2 == 1:
            return float(find_kth(total // 2 + 1))
        return (find_kth(total // 2) + find_kth(total // 2 + 1)) / 2.0
```

## 解法三：二分划分（推荐，Hard 标准答案）

### 思路

在较短数组 `nums1` 上二分划分点 `i`（左半取 i 个），`nums2` 左半取 `j = total_left - i` 个，使左半总个数 `(m+n+1)//2`。

**合法划分**需满足：
- `maxLeft1 <= minRight2`
- `maxLeft2 <= minRight1`

其中边界用 ±∞ 处理空半区。

若不满足则调整二分边界：
- `maxLeft1 > minRight2` → i 太大，`hi = i - 1`
- 否则 `lo = i + 1`

### 代码

```python
from typing import List


class Solution:
    def findMedianSortedArrays(self, nums1: List[int], nums2: List[int]) -> float:
        if len(nums1) > len(nums2):
            nums1, nums2 = nums2, nums1
        m, n = len(nums1), len(nums2)
        total_left = (m + n + 1) // 2
        lo, hi = 0, m
        while lo <= hi:
            i = (lo + hi) // 2
            j = total_left - i
            max_left1 = float("-inf") if i == 0 else nums1[i - 1]
            min_right1 = float("inf") if i == m else nums1[i]
            max_left2 = float("-inf") if j == 0 else nums2[j - 1]
            min_right2 = float("inf") if j == n else nums2[j]
            if max_left1 <= min_right2 and max_left2 <= min_right1:
                if (m + n) % 2 == 1:
                    return float(max(max_left1, max_left2))
                return (max(max_left1, max_left2) + min(min_right1, min_right2)) / 2.0
            elif max_left1 > min_right2:
                hi = i - 1
            else:
                lo = i + 1
        return 0.0
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(log(min(m,n))) |
| 空间 | O(1) |

### 讲解

核心思想：**中位数由划分线决定，不必真正合并**。

想象把两数组排成一行，用一条线分成左右两半，左半全部 ≤ 右半。中位数只取决于划分线附近的四个数：`maxLeft1, minRight1, maxLeft2, minRight2`。

| 解法 | 时间 | 空间 | 说明 |

| 合并 | O(m+n) | O(m+n) | 入门 |
| 第 k 小 | O(log(m+n)) | O(1) | 分治 |
| 二分划分 | O(log(min(m,n))) | O(1) | 面试 Hard 经典 |

## 常见错误

1. **忘记交换使 nums1 更短**：二分应在较短数组上进行。
2. **奇偶长度处理**：奇数返回 `max(maxLeft1, maxLeft2)`；偶数再与两个 minRight 平均。
3. **空数组边界**：i=0 或 i=m 时用 ±∞ 代替缺失的 maxLeft/minRight。

## 举一反三

- **两个数组的第 K 个最大元素**：堆或分治
- **寻找两个正序数组的中位数**是划分型二分的巅峰题
