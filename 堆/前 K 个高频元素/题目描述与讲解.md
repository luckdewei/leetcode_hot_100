# 前 K 个高频元素（Top K Frequent Elements）

- 题目链接：https://leetcode.cn/problems/top-k-frequent-elements/
- 难度：中等
- 标签：数组、哈希表、堆、桶排序

## 题目描述

给你一个整数数组 `nums` 和一个整数 `k`，请你返回其中出现频率前 `k` 高的元素。可以按任意顺序返回答案。

### 示例

```
输入：nums = [1,1,1,2,2,3], k = 2
输出：[1,2]

输入：nums = [1], k = 1
输出：[1]
```

### 约束

- `1 <= nums.length <= 10^5`
- `-10^4 <= nums[i] <= 10^4`
- `k` 的取值范围 `[1, 不同元素个数]`
- 答案保证唯一

---

## 解法一：小根堆维护 Top K（推荐）

### 思路

先用 `Counter` 统计频次，再用大小为 `k` 的小根堆：堆顶是 k 个高频元素中频次最小的，新元素频次更高则替换堆顶。

### 代码

```python
from typing import List
import heapq
from collections import Counter


class Solution:
    def topKFrequent(self, nums: List[int], k: int) -> List[int]:
        count = Counter(nums)
        heap = []
        for num, freq in count.items():
            heapq.heappush(heap, (freq, num))
            if len(heap) > k:
                heapq.heappop(heap)
        return [num for _, num in heap]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n log k) |
| 空间 | O(n) |

### 讲解

堆大小恒为 k，总 push/pop 次数 O(n log k)。比全排序 O(n log n) 更优，是面试常用写法。

## 解法二：桶排序

### 思路

频次最大不超过 n，建 n+1 个桶，`bucket[freq]` 存频次为 freq 的所有数，从高到低取 k 个。

### 代码

```python
from typing import List
from collections import Counter


class Solution:
    def topKFrequent(self, nums: List[int], k: int) -> List[int]:
        count = Counter(nums)
        buckets = [[] for _ in range(len(nums) + 1)]
        for num, freq in count.items():
            buckets[freq].append(num)
        ans = []
        for freq in range(len(buckets) - 1, 0, -1):
            for num in buckets[freq]:
                ans.append(num)
                if len(ans) == k:
                    return ans
        return ans
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(n) |

### 讲解

频次范围有界时可达 O(n)，比堆更快。但空间与 n 成正比，极端频次大时桶数组很长。

## 解法三：全排序

### 思路

统计频次后按频次降序排序，取前 k 个。

### 代码

```python
from typing import List
from collections import Counter


class Solution:
    def topKFrequent(self, nums: List[int], k: int) -> List[int]:
        count = Counter(nums)
        sorted_items = sorted(count.items(), key=lambda x: -x[1])
        return [num for num, _ in sorted_items[:k]]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n log n) |
| 空间 | O(n) |

### 讲解

最直观，不同元素个数远小于 n 时仍然可接受，但不如堆和桶排序优。

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 小根堆 | O(n log k) | O(n) | 面试首选，k 较小 |
| 桶排序 | O(n) | O(n) | 追求最优时间 |
| 全排序 | O(n log n) | O(n) | 写法最简单 |

---

## 常见错误

1. **用大根堆却维护了 k 个最小元素**：应维护小根堆，堆顶是 k 高频中最小的。
2. **返回顺序纠结**：题目允许任意顺序。
3. **与「数组中的第 K 个最大元素」混淆**：215 题找第 k 大值，本题找频次前 k。
