# 跳跃游戏 II（Jump Game II）

- 题目链接：https://leetcode.cn/problems/jump-game-ii/
- 难度：中等
- 标签：贪心、数组、动态规划

## 题目描述

给定 0 下标非负整数数组 `nums`，你最初位于下标 0。每个元素 `nums[i]` 表示从下标 i 向前最多跳 `nums[i]` 步。返回到达 `n - 1` 的**最少跳跃次数**。题目保证可以到达。

### 示例

```
输入：nums = [2,3,1,1,4]
输出：2
解释：0→1→4

输入：nums = [2,3,0,1,4]
输出：2
```

### 约束

- `1 <= nums.length <= 10^4`
- `0 <= nums[i] <= 1000`
- 保证可以到达 `n - 1`

---

## 解法一：贪心按层扩展（推荐）

### 思路

维护当前步数能到达的边界 `cur_end` 和下一步最远位置 `farthest`。遍历到 `cur_end` 时步数 +1，并更新 `cur_end = farthest`。

### 代码

```python
from typing import List


class Solution:
    def jump(self, nums: List[int]) -> int:
        steps = 0
        cur_end = 0
        farthest = 0
        n = len(nums)
        for i in range(n - 1):
            farthest = max(farthest, i + nums[i])
            if i == cur_end:
                steps += 1
                cur_end = farthest
        return steps
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(1) |

### 讲解

类似 BFS 按层计数，但 O(n) 一次遍历。循环到 n-2 即可，到达 n-1 前已计数。

## 解法二：BFS

### 思路

每层扩展当前可达的所有位置，层数即步数。

### 代码

```python
from typing import List
from collections import deque


class Solution:
    def jump(self, nums: List[int]) -> int:
        n = len(nums)
        if n <= 1:
            return 0
        q = deque([0])
        steps = 0
        visited = {0}
        while q:
            steps += 1
            for _ in range(len(q)):
                i = q.popleft()
                for j in range(i + 1, min(i + nums[i] + 1, n)):
                    if j == n - 1:
                        return steps
                    if j not in visited:
                        visited.add(j)
                        q.append(j)
        return steps
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n²) 最坏 |
| 空间 | O(n) |

### 讲解

BFS 保证最少步，但最坏 O(n²)，不如贪心优。

## 解法三：动态规划

### 思路

`dp[i]` 为到达 i 的最少步数，`dp[i] = min(dp[j] + 1)` 对所有 `j + nums[j] >= i` 且 `j < i`。

### 代码

```python
from typing import List


class Solution:
    def jump(self, nums: List[int]) -> int:
        n = len(nums)
        dp = [float("inf")] * n
        dp[0] = 0
        for i in range(1, n):
            for j in range(i):
                if j + nums[j] >= i:
                    dp[i] = min(dp[i], dp[j] + 1)
        return dp[n - 1]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n²) |
| 空间 | O(n) |

### 讲解

O(n²) DP，直观但效率不如贪心。

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 贪心按层 | O(n) | O(1) | 面试首选 |
| BFS | O(n²) | O(n) | 理解层次 |
| DP | O(n²) | O(n) | 过渡理解 |

## 常见错误

1. **循环到 n-1**：应到 n-2，最后一步不计入多余层。
2. **与 Jump Game I 混淆**：I 题只问能否，II 题求最少步。
3. **贪心边界更新时机错**：应在 `i == cur_end` 时步数+1。
