# 跳跃游戏（Jump Game）

- 题目链接：https://leetcode.cn/problems/jump-game/
- 难度：中等
- 标签：贪心、数组、动态规划

## 题目描述

给定非负整数数组 `nums`，你最初位于第一个下标。数组中每个元素代表在该位置可跳跃的最大长度。判断是否能够到达最后一个下标。

### 示例

```
输入：nums = [2,3,1,1,4]
输出：true
解释：0→1→4

输入：nums = [3,2,1,0,4]
输出：false
```

### 约束

- `1 <= nums.length <= 10^4`
- `0 <= nums[i] <= 10^5`

---

## 解法一：维护最远可达位置（推荐）

### 思路

遍历数组，维护 `reach` 为目前能到达的最远下标。若 `i > reach` 则无法到达 i；否则更新 `reach = max(reach, i + nums[i])`。

### 代码

```python
from typing import List


class Solution:
    def canJump(self, nums: List[int]) -> bool:
        reach = 0
        for i, jump in enumerate(nums):
            if i > reach:
                return False
            reach = max(reach, i + jump)
        return True
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(1) |

### 讲解

只需知道最远可达，不必记录具体路径。到达最后一个下标即 `reach >= n-1`（循环自然保证）。

---

## 解法二：逆向贪心

### 思路

从末尾向前，维护 `goal` 为当前需要到达的目标下标，若 `i + nums[i] >= goal` 则更新 `goal = i`。最终 `goal == 0` 即可达。

### 代码

```python
from typing import List


class Solution:
    def canJump(self, nums: List[int]) -> bool:
        goal = len(nums) - 1
        for i in range(len(nums) - 2, -1, -1):
            if i + nums[i] >= goal:
                goal = i
        return goal == 0
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(1) |

### 讲解

逆向思维：能否到达终点等价于能否从某点跳到当前 goal，逐步前移 goal。

## 解法三：动态规划（理解用）

### 思路

`dp[i]` 表示能否到达 i，转移：`dp[j] = True` 且 `j + nums[j] >= i` 则 `dp[i] = True`。

### 代码

```python
from typing import List


class Solution:
    def canJump(self, nums: List[int]) -> bool:
        n = len(nums)
        dp = [False] * n
        dp[0] = True
        for i in range(n):
            if not dp[i]:
                continue
            for j in range(1, nums[i] + 1):
                if i + j < n:
                    dp[i + j] = True
        return dp[n - 1]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n²) 最坏 |
| 空间 | O(n) |

### 讲解

O(n²) 可能超时，但帮助理解可达性 DP。

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 最远可达 | O(n) | O(1) | 面试首选 |
| 逆向贪心 | O(n) | O(1) | 替代写法 |
| DP | O(n²) | O(n) | 理解用 |

## 常见错误

1. **忘记判断 `i > reach`**：当前位置不可达应直接返回 False。
2. **与 Jump Game II 混淆**：II 题求最少步数，本题只问能否到达。
3. **BFS 层数当步数**：I 题不需要最少步。
