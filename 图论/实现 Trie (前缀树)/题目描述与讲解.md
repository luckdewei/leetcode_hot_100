# 实现 Trie (前缀树)（Implement Trie）

- 题目链接：https://leetcode.cn/problems/implement-trie-prefix-tree/
- 难度：中等
- 标签：设计、字典树、哈希表、字符串

## 题目描述

**Trie**（发音类似 "try"）即字典树，又称前缀树或单词查找树。

实现 Trie 类：

- `Trie()` 初始化前缀树对象。
- `void insert(String word)` 向前缀树中插入字符串 `word`。
- `boolean search(String word)` 如果字符串 `word` 在前缀树中，返回 `true`；否则返回 `false`（必须完全匹配，不能是其他词的前缀）。
- `boolean startsWith(String prefix)` 如果之前已插入的字符串 `word` 的前缀之一为 `prefix`，返回 `true`；否则返回 `false`。

### 示例

```
输入：
["Trie", "insert", "search", "search", "startsWith", "insert", "search"]
[[], ["apple"], ["apple"], ["app"], ["app"], ["app"], ["app"]]
输出：[null, null, true, false, true, null, true]

解释：
Trie trie = new Trie();
trie.insert("apple");
trie.search("apple");   // true
trie.search("app");     // false（未插入完整单词 "app"）
trie.startsWith("app"); // true
trie.insert("app");
trie.search("app");     // true
```

### 约束

- `1 <= word.length, prefix.length <= 2000`
- `word` 和 `prefix` 仅由小写英文字母组成
- `insert`、`search`、`startsWith` 调用次数总计不超过 `3 × 10^4`

## 解法一：字典树节点 + 哈希子节点（推荐）

### 思路

每个节点维护 `children` 字典（26 个字母）和 `is_end` 标记单词结尾。插入沿路径建节点；搜索需走到末尾且 `is_end`；前缀查询只需路径存在。

### 代码

```python
class TrieNode:
    def __init__(self):
        self.children = {}
        self.is_end = False


class Trie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, word: str) -> None:
        node = self.root
        for ch in word:
            if ch not in node.children:
                node.children[ch] = TrieNode()
            node = node.children[ch]
        node.is_end = True

    def search(self, word: str) -> bool:
        node = self._find(word)
        return node is not None and node.is_end

    def startsWith(self, prefix: str) -> bool:
        return self._find(prefix) is not None

    def _find(self, s: str):
        node = self.root
        for ch in s:
            if ch not in node.children:
                return None
            node = node.children[ch]
        return node
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | 插入/查询/前缀 O(L)，L 为字符串长度 |
| 空间 | O(N × L)，N 为插入单词数，均摊总字符数 |

### 讲解

面试标准实现。哈希表 `children` 灵活；固定 26 数组略省哈希开销。

## 解法二：固定大小数组 `[26]` 子节点

### 思路

每个节点用长度为 26 的数组存子指针，`ord(ch)-ord('a')` 作下标。逻辑相同，常数略优。

### 代码

```python
class TrieNode:
    __slots__ = ("children", "is_end")
    def __init__(self):
        self.children = [None] * 26
        self.is_end = False


class Trie:
    def __init__(self):
        self.root = TrieNode()

    def _idx(self, ch: str) -> int:
        return ord(ch) - ord("a")

    def insert(self, word: str) -> None:
        node = self.root
        for ch in word:
            i = self._idx(ch)
            if node.children[i] is None:
                node.children[i] = TrieNode()
            node = node.children[i]
        node.is_end = True

    def search(self, word: str) -> bool:
        node = self._find(word)
        return node is not None and node.is_end

    def startsWith(self, prefix: str) -> bool:
        return self._find(prefix) is not None

    def _find(self, s: str):
        node = self.root
        for ch in s:
            i = self._idx(ch)
            if node.children[i] is None:
                return None
            node = node.children[i]
        return node
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(L) 各操作 |
| 空间 | O(N × L)，每节点固定 26 指针 |

### 讲解

仅小写字母时数组更快。`__slots__` 可略减内存。

## 解法三：哈希表暴力存储（对比用）

### 思路

用 `set` 存所有插入单词，`search` 查 set，`startsWith` 遍历 set 看是否有以 prefix 开头的词。能过部分测试但效率差。

### 代码

```python
class Trie:
    def __init__(self):
        self.words = set()

    def insert(self, word: str) -> None:
        self.words.add(word)

    def search(self, word: str) -> bool:
        return word in self.words

    def startsWith(self, prefix: str) -> bool:
        return any(w.startswith(prefix) for w in self.words)
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | search O(L)，startsWith O(N × L) |
| 空间 | O(N × L) |

### 讲解

说明 Trie 的价值：前缀查询 O(L)，而非 O(N × L)。数据量大时暴力超时。

## 常见错误

1. **search 与 startsWith 混淆**：search 需 `is_end`，startsWith 不需要。
2. **insert 未设 is_end**：短词被长词前缀覆盖时 search 错误。
3. **空字符串**：插入 `""` 时根节点 `is_end=True`，需正确处理。
4. **与「单词搜索 II」混淆**：212 题在网格上用 Trie 搜多词，208 题是纯数据结构。
