# 课程表（Course Schedule）

- 题目链接：https://leetcode.cn/problems/course-schedule/
- 难度：中等
- 标签：深度优先搜索、广度优先搜索、图、拓扑排序

## 题目描述

你这个学期有 `numCourses` 门课，编号为 `0` 到 `numCourses - 1`。给你一个数组 `prerequisites`，其中 `prerequisites[i] = [a_i, b_i]` 表示如果要学习课程 `a_i` 则**必须先**学习课程 `b_i`。

例如，先修课程对 `[0, 1]` 表示：想要学习课程 `0`，你需要先完成课程 `1`。

请你判断是否可能完成所有课程的学习？

### 示例

```
输入：numCourses = 2, prerequisites = [[1,0]]
输出：true
解释：共 2 门课，先修 0 再学 1，可以完成。

输入：numCourses = 2, prerequisites = [[1,0],[0,1]]
输出：false
解释：0 依赖 1，1 依赖 0，形成环，无法完成。
```

### 约束

- `1 <= numCourses <= 2000`
- `0 <= prerequisites.length <= 5000`
- `prerequisites[i].length == 2`
- `0 <= a_i, b_i < numCourses`
- `prerequisites[i]` 中所有 `(a_i, b_i)` **互不相同**

---

## 解法一：拓扑排序 BFS / Kahn 算法（推荐）

### 思路

建图：`b → a` 表示 b 是 a 的前置。统计入度，入度为 0 的课入队。每取出一门课，其邻居入度 -1，变为 0 则入队。最终 `taken == numCourses` 则无环。

### 代码

```python
from collections import deque
from typing import List


class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        graph = [[] for _ in range(numCourses)]
        indeg = [0] * numCourses
        for a, b in prerequisites:
            graph[b].append(a)
            indeg[a] += 1

        q = deque(i for i in range(numCourses) if indeg[i] == 0)
        taken = 0
        while q:
            u = q.popleft()
            taken += 1
            for v in graph[u]:
                indeg[v] -= 1
                if indeg[v] == 0:
                    q.append(v)
        return taken == numCourses
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(V + E)，V 为课程数，E 为先修关系数 |
| 空间 | O(V + E) |

### 讲解

有向图判环的标准解法之一。BFS 拓扑序与 DFS 三色标记等价，BFS 更直观。

## 解法二：DFS 三色标记判环

### 思路

`0=未访问, 1=访问中, 2=已完成`。DFS 遇到状态 1 的节点说明有环。全部 DFS 结束无环则可完成。

### 代码

```python
from typing import List


class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        graph = [[] for _ in range(numCourses)]
        for a, b in prerequisites:
            graph[b].append(a)

        state = [0] * numCourses  # 0 white, 1 gray, 2 black

        def dfs(u: int) -> bool:
            if state[u] == 1:
                return False
            if state[u] == 2:
                return True
            state[u] = 1
            for v in graph[u]:
                if not dfs(v):
                    return False
            state[u] = 2
            return True

        return all(dfs(i) for i in range(numCourses))
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(V + E) |
| 空间 | O(V + E) |

### 讲解

DFS 判环经典模板。`gray` 表示当前递归栈内，遇到 gray 即回边/环。

## 解法三：拓扑排序 DFS（后序栈）

### 思路

DFS 完成后将节点压栈，逆序即拓扑序。若栈长 < numCourses 则有环（或等价于 DFS 中发现环）。

### 代码

```python
from typing import List


class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        graph = [[] for _ in range(numCourses)]
        for a, b in prerequisites:
            graph[b].append(a)

        visited = [0] * numCourses
        order = []

        def dfs(u: int) -> bool:
            if visited[u] == 1:
                return False
            if visited[u] == 2:
                return True
            visited[u] = 1
            for v in graph[u]:
                if not dfs(v):
                    return False
            visited[u] = 2
            order.append(u)
            return True

        for i in range(numCourses):
            if visited[i] == 0 and not dfs(i):
                return False
        return len(order) == numCourses
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(V + E) |
| 空间 | O(V + E) |

### 讲解

与解法二本质相同，额外记录拓扑序。**课程表 II**（210）需输出具体顺序，用此思路。

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| Kahn BFS | O(V+E) | O(V+E) | 面试首选 |
| DFS 三色 | O(V+E) | O(V+E) | 判环通用 |
| DFS 拓扑序 | O(V+E) | O(V+E) | 需输出顺序时 |

## 常见错误

1. **建边方向反了**：`[a,b]` 应是 `b → a`，不是 `a → b`。
2. **只从节点 0 DFS**：图可能不连通，需遍历所有节点。
3. **入度统计遗漏**：每条边只给 `a` 加一次入度。
4. **与「并查集」误用**：并查集判无向环，有向环需拓扑/DFS。
