# 腐烂的橘子（Rotting Oranges）

- 题目链接：https://leetcode.cn/problems/rotting-oranges/
- 难度：中等
- 标签：广度优先搜索、数组、矩阵

## 题目描述

在给定的 `m × n` 网格 `grid` 中，每个单元格可以有以下三个值之一：

- `0` 表示空位置
- `1` 表示新鲜橘子
- `2` 表示腐烂的橘子

每分钟，腐烂的橘子周围 **4 个方向上** 相邻的新鲜橘子都会腐烂。

返回直到单元格中没有新鲜橘子为止所必须经过的**最小分钟数**。如果不可能，返回 `-1`。

### 示例

```
输入：grid = [[2,1,1],[1,1,0],[0,1,1]]
输出：4

输入：grid = [[2,1,1],[0,1,1],[1,0,1]]
输出：-1

输入：grid = [[0,2]]
输出：0
```

### 约束

- `m == grid.length`
- `n == grid[i].length`
- `1 <= m, n <= 10`
- `grid[i][j]` 为 `0`、`1` 或 `2`

---

## 解法一：多源 BFS（推荐）

### 思路

先将所有腐烂橘子 `(2)` 入队，同时统计新鲜橘子数 `fresh`。BFS 按层扩散（每层 = 1 分钟），感染相邻新鲜橘子。队列为空后，若 `fresh == 0` 返回分钟数，否则 `-1`。

### 代码

```python
from collections import deque
from typing import List


class Solution:
    def orangesRotting(self, grid: List[List[int]]) -> int:
        m, n = len(grid), len(grid[0])
        q = deque()
        fresh = 0
        for i in range(m):
            for j in range(n):
                if grid[i][j] == 2:
                    q.append((i, j))
                elif grid[i][j] == 1:
                    fresh += 1
        if fresh == 0:
            return 0

        minutes, dirs = 0, [(0, 1), (0, -1), (1, 0), (-1, 0)]
        while q:
            for _ in range(len(q)):
                r, c = q.popleft()
                for dr, dc in dirs:
                    nr, nc = r + dr, c + dc
                    if 0 <= nr < m and 0 <= nc < n and grid[nr][nc] == 1:
                        grid[nr][nc] = 2
                        fresh -= 1
                        q.append((nr, nc))
            if q:
                minutes += 1
        return minutes if fresh == 0 else -1
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(mn) |
| 空间 | O(mn)，队列 |

### 讲解

多源 BFS 经典题：所有起点同时入队，层数即时间。注意 `if q: minutes += 1` 避免最后一层空转。

## 解法二：单源 BFS 逐个腐烂（理解用）

### 思路

每分钟扫描全网格，找能感染的新鲜橘子。逻辑正确但重复扫描，效率低。

### 代码

```python
from typing import List


class Solution:
    def orangesRotting(self, grid: List[List[int]]) -> int:
        m, n = len(grid), len(grid[0])
        dirs = [(0, 1), (0, -1), (1, 0), (-1, 0)]
        minutes = 0

        while True:
            to_rot = []
            fresh = 0
            for i in range(m):
                for j in range(n):
                    if grid[i][j] == 1:
                        fresh += 1
                    if grid[i][j] != 2:
                        continue
                    for dr, dc in dirs:
                        ni, nj = i + dr, j + dc
                        if 0 <= ni < m and 0 <= nj < n and grid[ni][nj] == 1:
                            to_rot.append((ni, nj))
            if not to_rot:
                return minutes if fresh == 0 else -1
            for i, j in to_rot:
                grid[i][j] = 2
            minutes += 1
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O((mn)²) 最坏 |
| 空间 | O(mn) |

### 讲解

帮助理解「按分钟模拟」，但应改用多源 BFS。

## 解法三：DFS 逐源扩散（不推荐）

### 思路

从每个腐烂点 DFS 记录到达每个新鲜橘子的最早时间，取全局最大值。多源时需取 min 时间，实现繁琐，易错。

### 代码

```python
from typing import List


class Solution:
    def orangesRotting(self, grid: List[List[int]]) -> int:
        m, n = len(grid), len(grid[0])
        INF = 10 ** 9
        dist = [[INF] * n for _ in range(m)]
        q = []

        for i in range(m):
            for j in range(n):
                if grid[i][j] == 2:
                    dist[i][j] = 0
                    q.append((0, i, j))
                elif grid[i][j] == 0:
                    dist[i][j] = 0

        dirs = [(0, 1), (0, -1), (1, 0), (-1, 0)]
        head = 0
        while head < len(q):
            t, r, c = q[head]
            head += 1
            for dr, dc in dirs:
                nr, nc = r + dr, c + dc
                if 0 <= nr < m and 0 <= nc < n and t + 1 < dist[nr][nc]:
                    dist[nr][nc] = t + 1
                    q.append((t + 1, nr, nc))

        ans, has_fresh = 0, False
        for i in range(m):
            for j in range(n):
                if grid[i][j] == 1:
                    has_fresh = True
                    if dist[i][j] == INF:
                        return -1
                    ans = max(ans, dist[i][j])
        return ans if has_fresh else 0
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(mn) |
| 空间 | O(mn) |

### 讲解

本质是带权 BFS（边权为 1）。多源 BFS 写法二更简洁；此写法类似 Dijkstra 在边权为 1 时的退化。

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 多源 BFS | O(mn) | O(mn) | 面试首选 |
| 逐分钟扫描 | O((mn)²) | O(mn) | 理解模拟 |
| 最短时间 BFS | O(mn) | O(mn) | 求各点最早感染时间 |

---

## 常见错误

1. **fresh 初始为 0 未提前返回**：无新鲜橘子时应返回 0 而非继续 BFS。
2. **minutes 多计一层**：最后一轮队列为空时不应再 +1。
3. **对角线感染**：只能四方向。
4. **与「01 矩阵」混淆**：542 题是 0/1 最短路径，也是 BFS。
