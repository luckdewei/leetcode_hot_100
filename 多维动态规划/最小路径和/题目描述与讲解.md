# 最小路径和（Minimum Path Sum）

- 题目链接：https://leetcode.cn/problems/minimum-path-sum/
- 难度：中等
- 标签：数组、动态规划、矩阵

## 题目描述

给定一个包含非负整数的 `m × n` 网格 `grid`，请找出一条从左上角到右下角的路径，使得路径上的数字总和为最小。

**说明**：每次只能向下或者向右移动一步。

### 示例

```
输入：grid = [[1,3,1],[1,5,1],[4,2,1]]
输出：7
解释：路径 1→3→1→1→1，总和 7

输入：grid = [[1,2,3],[4,5,6]]
输出：12
```

### 约束

- `m == grid.length`
- `n == grid[i].length`
- `1 <= m, n <= 200`
- `0 <= grid[i][j] <= 200`

---

## 解法一：二维 DP（原地修改 grid）

### 思路

1. `dp[i][j]`：到 (i,j) 的最小路径和。
2. 第一行累加左边，第一列累加上边。
3. `dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1])`。
4. 可直接修改 `grid` 省空间。

### 代码

```python
from typing import List


class Solution:
    def minPathSum(self, grid: List[List[int]]) -> int:
        m, n = len(grid), len(grid[0])
        for i in range(m):
            for j in range(n):
                if i == 0 and j == 0:
                    continue
                if i == 0:
                    grid[i][j] += grid[i][j - 1]
                elif j == 0:
                    grid[i][j] += grid[i - 1][j]
                else:
                    grid[i][j] += min(grid[i - 1][j], grid[i][j - 1])
        return grid[m - 1][n - 1]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(m × n) |
| 空间 | O(1) 额外（改原数组） |

### 讲解

与不同路径结构相同，只是把「加路径数」换成「取 min 加代价」。原地修改简洁但破坏输入。

## 解法二：一维滚动（推荐，与 solution.py 一致）

### 思路

1. `dp[j]` 表示当前行到列 j 的最小和。
2. 初始化第一行前缀和；逐行更新：`dp[j] = grid[i][j] + min(dp[j], dp[j-1])`。
3. 第一列每行单独累加 `dp[0]`。

### 代码

```python
from typing import List


class Solution:
    def minPathSum(self, grid: List[List[int]]) -> int:
        m, n = len(grid), len(grid[0])
        dp = grid[0][:]
        for j in range(1, n):
            dp[j] += dp[j - 1]
        for i in range(1, m):
            dp[0] += grid[i][0]
            for j in range(1, n):
                dp[j] = grid[i][j] + min(dp[j], dp[j - 1])
        return dp[-1]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(m × n) |
| 空间 | O(n) |

### 讲解

与 `solution.py` 一致。不修改原 grid，面试更稳妥。注意每行开始时更新 `dp[0]`。

## 解法三：记忆化 DFS

### 思路

`dfs(i,j)` 为到 (i,j) 的最小和，从 (0,0) 递归到 (m-1,n-1)，加 memo。

### 代码

```python
from typing import List
from functools import lru_cache


class Solution:
    def minPathSum(self, grid: List[List[int]]) -> int:
        m, n = len(grid), len(grid[0])

        @lru_cache(maxsize=None)
        def dfs(i: int, j: int) -> int:
            if i == 0 and j == 0:
                return grid[0][0]
            if i == 0:
                return dfs(i, j - 1) + grid[i][j]
            if j == 0:
                return dfs(i - 1, j) + grid[i][j]
            return grid[i][j] + min(dfs(i - 1, j), dfs(i, j - 1))

        return dfs(m - 1, n - 1)
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(m × n) |
| 空间 | O(m × n) |

### 讲解

自顶向下与迭代 DP 等价，大数据可能递归栈较深，优先迭代。

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 二维原地 | O(mn) | O(1) | 可改输入 |
| 一维滚动 | O(mn) | O(n) | 面试首选 |
| 记忆化 | O(mn) | O(mn) | 理解递归 |

## 常见错误

1. **第一行/列初始化**：不能只设 `dp[0][0]`，要整行整列前缀累加。
2. **与不同路径混淆**：本题求 min sum，不是 path count。
3. **负数**：本题非负，若有负数需标准 DP 不能贪心。
