# 最长回文子串（Longest Palindromic Substring）

- 题目链接：https://leetcode.cn/problems/longest-palindromic-substring/
- 难度：中等
- 标签：双指针、字符串、动态规划

## 题目描述

给你一个字符串 `s`，找到 `s` 中最长的回文子串。

### 示例

```
输入：s = "babad"
输出："bab"
解释："aba" 同样是符合题意的答案

输入：s = "cbbd"
输出："bb"
```

### 约束

- `1 <= s.length <= 1000`
- `s` 仅由数字和英文字母组成

---

## 解法一：中心扩展（推荐，与 solution.py 一致）

### 思路

1. 回文串有奇、偶两种中心：单字符 `(i,i)` 或双字符 `(i,i+1)`。
2. 从每个中心向两侧扩展，直到字符不相等。
3. 记录最长回文的起始位置和长度，最后切片返回。

### 代码

```python
class Solution:
    def longestPalindrome(self, s: str) -> str:
        n = len(s)
        if n < 2:
            return s
        start = max_len = 0

        def expand(l: int, r: int):
            nonlocal start, max_len
            while l >= 0 and r < n and s[l] == s[r]:
                l -= 1
                r += 1
            length = r - l - 1
            if length > max_len:
                max_len = length
                start = l + 1

        for i in range(n):
            expand(i, i)
            expand(i, i + 1)
        return s[start : start + max_len]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n²) |
| 空间 | O(1) |

### 讲解

与 `solution.py` 一致。扩展结束后 `l+1` 到 `r-1` 为回文，长度 `r-l-1`。面试最推荐的平衡写法。

## 解法二：动态规划

### 思路

1. `dp[i][j]`：子串 `s[i:j+1]` 是否回文。
2. 若 `s[i]==s[j]` 且（长度≤2 或 `dp[i+1][j-1]`），则 `dp[i][j]=True`。
3. 按子串长度从小到大填表，更新最长答案。

### 代码

```python
class Solution:
    def longestPalindrome(self, s: str) -> str:
        n = len(s)
        if n < 2:
            return s
        dp = [[False] * n for _ in range(n)]
        start = max_len = 0
        for i in range(n):
            dp[i][i] = True
        for length in range(2, n + 1):
            for i in range(n - length + 1):
                j = i + length - 1
                if s[i] != s[j]:
                    continue
                if length <= 2 or dp[i + 1][j - 1]:
                    dp[i][j] = True
                    if length > max_len:
                        max_len = length
                        start = i
        return s[start : start + max_len]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n²) |
| 空间 | O(n²) |

### 讲解

经典二维 DP，与「回文子串计数」等问题框架一致。空间较大，但逻辑清晰。

## 解法三：Manacher 算法（进阶）

### 思路

在 O(n) 内求最长回文：预处理插入 `#` 分隔符，维护最右回文边界和中心，利用对称性跳过已知区域。

### 代码

```python
class Solution:
    def longestPalindrome(self, s: str) -> str:
        t = "#".join(s)
        n = len(t)
        p = [0] * n
        center = right = 0
        best_center = best_len = 0
        for i in range(n):
            mirror = 2 * center - i
            if i < right:
                p[i] = min(right - i, p[mirror])
            while i - p[i] - 1 >= 0 and i + p[i] + 1 < n and t[i - p[i] - 1] == t[i + p[i] + 1]:
                p[i] += 1
            if i + p[i] > right:
                center, right = i, i + p[i]
            if p[i] > best_len:
                best_len = p[i]
                best_center = i
        start = (best_center - best_len) // 2
        return s[start : start + best_len]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(n) |

### 讲解

竞赛级算法，Hot 100 面试中中心扩展足够。了解 Manacher 可应对 follow-up「能否 O(n)」。

## 常见错误

1. **偶数长度中心**：必须尝试 `(i,i+1)`，否则漏掉 `"bb"`。
2. **扩展边界**：while 内先比较再扩展，结束后长度是 `r-l-1`。
3. **与 647 题混淆**：647 求回文子串个数，本题求最长子串本身。
