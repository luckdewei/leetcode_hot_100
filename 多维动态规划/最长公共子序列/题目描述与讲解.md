# 最长公共子序列（Longest Common Subsequence）

- 题目链接：https://leetcode.cn/problems/longest-common-subsequence/
- 难度：中等
- 标签：字符串、动态规划

## 题目描述

给定两个字符串 `text1` 和 `text2`，返回这两个字符串的最长**公共子序列**的长度。如果不存在公共子序列，返回 0。

**子序列** 是由原字符串删除一些（也可以不删除）字符而不改变相对顺序得到的新字符串。

例如，`"ace"` 是 `"abcde"` 的子序列，但 `"aec"` 不是。

### 示例

```
输入：text1 = "abcde", text2 = "ace"
输出：3
解释：最长公共子序列是 "ace"，长度为 3

输入：text1 = "abc", text2 = "abc"
输出：3

输入：text1 = "abc", text2 = "def"
输出：0
```

### 约束

- `1 <= text1.length, text2.length <= 1000`
- `text1` 和 `text2` 仅由小写英文字符组成

---

## 解法一：二维 DP（推荐，与 solution.py 一致）

### 思路

1. `dp[i][j]`：`text1[0:i]` 与 `text2[0:j]` 的 LCS 长度。
2. 若 `text1[i-1] == text2[j-1]`：`dp[i][j] = dp[i-1][j-1] + 1`。
3. 否则：`dp[i][j] = max(dp[i-1][j], dp[i][j-1])`。
4. 答案 `dp[m][n]`，多一行一列 0 边界简化边界。

### 代码

```python
class Solution:
    def longestCommonSubsequence(self, text1: str, text2: str) -> int:
        m, n = len(text1), len(text2)
        dp = [[0] * (n + 1) for _ in range(m + 1)]
        for i in range(1, m + 1):
            for j in range(1, n + 1):
                if text1[i - 1] == text2[j - 1]:
                    dp[i][j] = dp[i - 1][j - 1] + 1
                else:
                    dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])
        return dp[m][n]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(m × n) |
| 空间 | O(m × n) |

### 讲解

与 `solution.py` 一致。LCS 是二维 DP 模板题，匹配则斜对角 +1，不匹配则取上/左较大值。

## 解法二：空间优化为一维

### 思路

`dp[i][j]` 只依赖上一行和当前行，用一维数组从右到左更新：`dp[j] = dp[j-1]+1`（匹配）或 `max(dp[j], dp[j-1])`（不匹配，注意遍历顺序）。

匹配时更简洁写法：倒序 j，用 `prev` 存 `dp[j-1]` 旧值。

### 代码

```python
class Solution:
    def longestCommonSubsequence(self, text1: str, text2: str) -> int:
        m, n = len(text1), len(text2)
        dp = [0] * (n + 1)
        for i in range(1, m + 1):
            prev = 0
            for j in range(1, n + 1):
                temp = dp[j]
                if text1[i - 1] == text2[j - 1]:
                    dp[j] = prev + 1
                else:
                    dp[j] = max(dp[j], dp[j - 1])
                prev = temp
        return dp[n]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(m × n) |
| 空间 | O(n) |

### 讲解

面试若写完二维可主动优化空间。`prev` 保存更新前的 `dp[j-1]`，等价于二维的 `dp[i-1][j-1]`。

## 解法三：记忆化 DFS

### 思路

`dfs(i,j)` 表示 `text1[i:]` 与 `text2[j:]` 的 LCS 长度，字符相等则 +1 并双指针前进，否则取 `max(dfs(i+1,j), dfs(i,j+1))`。

### 代码

```python
from functools import lru_cache


class Solution:
    def longestCommonSubsequence(self, text1: str, text2: str) -> int:
        @lru_cache(maxsize=None)
        def dfs(i: int, j: int) -> int:
            if i == len(text1) or j == len(text2):
                return 0
            if text1[i] == text2[j]:
                return 1 + dfs(i + 1, j + 1)
            return max(dfs(i + 1, j), dfs(i, j + 1))

        return dfs(0, 0)
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(m × n) |
| 空间 | O(m × n) |

### 讲解

与迭代 DP 状态一一对应，适合从递归理解「选匹配还是跳过」。

## 常见错误

1. **下标偏移**：字符串比较用 `[i-1]` 对应 `dp[i]`。
2. **子序列 vs 子串**：不要求连续，"ace" 在 "abcde" 中合法。
3. **与 1143 题重复**：本题即 LCS 标准题，勿与编辑距离混淆。
