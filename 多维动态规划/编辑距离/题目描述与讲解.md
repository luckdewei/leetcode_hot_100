# 编辑距离（Edit Distance）

- 题目链接：https://leetcode.cn/problems/edit-distance/
- 难度：中等
- 标签：字符串、动态规划

## 题目描述

给你两个单词 `word1` 和 `word2`，请返回将 `word1` 转换成 `word2` 所使用的最少操作数。

你可以对一个单词进行如下三种操作：

- 插入一个字符
- 删除一个字符
- 替换一个字符

### 示例

```
输入：word1 = "horse", word2 = "ros"
输出：3
解释：horse →rorse → rose → ros

输入：word1 = "intention", word2 = "execution"
输出：5
```

### 约束

- `0 <= word1.length, word2.length <= 500`
- `word1` 和 `word2` 由小写英文字母组成

---

## 解法一：二维 DP（推荐，与 solution.py 一致）

### 思路

1. `dp[i][j]`：将 `word1[0:i]` 变为 `word2[0:j]` 的最少操作数。
2. 初始化：`dp[i][0]=i`（全删），`dp[0][j]=j`（全插）。
3. 若 `word1[i-1]==word2[j-1]`：`dp[i][j]=dp[i-1][j-1]`。
4. 否则：`dp[i][j]=1+min(删 word1[i-1], 插 word2[j-1], 替换)` 即 `min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1])`。

### 代码

```python
class Solution:
    def minDistance(self, word1: str, word2: str) -> int:
        m, n = len(word1), len(word2)
        dp = [[0] * (n + 1) for _ in range(m + 1)]
        for i in range(m + 1):
            dp[i][0] = i
        for j in range(n + 1):
            dp[0][j] = j
        for i in range(1, m + 1):
            for j in range(1, n + 1):
                if word1[i - 1] == word2[j - 1]:
                    dp[i][j] = dp[i - 1][j - 1]
                else:
                    dp[i][j] = 1 + min(dp[i - 1][j], dp[i][j - 1], dp[i - 1][j - 1])
        return dp[m][n]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(m × n) |
| 空间 | O(m × n) |

### 讲解

与 `solution.py` 一致。三种操作对应 dp 来自上方（删）、左方（插）、左上（替）。LCS 的升级版，必背模板。

## 解法二：空间优化一维 DP

### 思路

滚动一行，从左到右更新，用 `prev` 保存左上角的旧值。

### 代码

```python
class Solution:
    def minDistance(self, word1: str, word2: str) -> int:
        m, n = len(word1), len(word2)
        dp = list(range(n + 1))
        for i in range(1, m + 1):
            prev = dp[0]
            dp[0] = i
            for j in range(1, n + 1):
                temp = dp[j]
                if word1[i - 1] == word2[j - 1]:
                    dp[j] = prev
                else:
                    dp[j] = 1 + min(dp[j], dp[j - 1], prev)
                prev = temp
        return dp[n]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(m × n) |
| 空间 | O(n) |

### 讲解

与 LCS 一维优化类似。面试写完二维后主动提 O(n) 空间是加分项。

## 解法三：记忆化 DFS

### 思路

`dfs(i,j)` 为后缀 `word1[i:]` 到 `word2[j:]` 的编辑距离，字符相等则 `dfs(i+1,j+1)`，否则 1 + min(三种操作)。

### 代码

```python
from functools import lru_cache


class Solution:
    def minDistance(self, word1: str, word2: str) -> int:
        @lru_cache(maxsize=None)
        def dfs(i: int, j: int) -> int:
            if i == len(word1):
                return len(word2) - j
            if j == len(word2):
                return len(word1) - i
            if word1[i] == word2[j]:
                return dfs(i + 1, j + 1)
            return 1 + min(
                dfs(i + 1, j),      # 删
                dfs(i, j + 1),      # 插
                dfs(i + 1, j + 1),  # 替
            )

        return dfs(0, 0)
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(m × n) |
| 空间 | O(m × n) |

### 讲解

边界 `i==m` 时剩余全插入，代价 `len(word2)-j`。递归版便于理解三种操作的含义。

| 二维 DP | O(mn) | O(mn) | 面试首选 |
| 一维 DP | O(mn) | O(n) | 空间优化 |
| 记忆化 | O(mn) | O(mn) | 理解操作语义 |

## 常见错误

1. **初始化**：空串到长度为 j 需 j 次插入，不是 0。
2. **匹配时不 +1**：相等时 `dp[i][j]=dp[i-1][j-1]`，无操作代价。
3. **与 LCS 混淆**：LCS 只求最长，编辑距离要 min 三种编辑代价。
