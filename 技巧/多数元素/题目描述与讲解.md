# 多数元素（Majority Element）

- 题目链接：https://leetcode.cn/problems/majority-element/
- 难度：简单
- 标签：数组、哈希表、分治、计数、排序

## 题目描述

给定一个大小为 `n` 的数组 `nums`，返回其中的多数元素。多数元素是指在数组中出现次数**大于** `⌊n/2⌋` 的元素。

你可以假设数组是非空的，并且数组中的多数元素总是存在。

### 示例

```
输入：nums = [3,2,3]
输出：3

输入：nums = [2,2,1,1,1,2,2]
输出：2
```

### 约束

- `n == nums.length`
- `1 <= n <= 5 × 10^4`
- `-10^9 <= nums[i] <= 10^9`
- 多数元素一定存在

---

## 解法一：哈希表计数

### 思路

1. 统计每个元素出现次数。
2. 返回次数 > n/2 的元素。

### 代码

```python
from typing import List
from collections import Counter


class Solution:
    def majorityElement(self, nums: List[int]) -> int:
        n = len(nums)
        for k, v in Counter(nums).items():
            if v > n // 2:
                return k
        return -1
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(n) |

### 讲解

正确且简单，但 O(n) 空间。面试应继续给出 Boyer-Moore。

---

## 解法二：Boyer-Moore 投票（推荐，与 solution.py 一致）

### 思路

1. 维护候选 `cand` 和票数 `count`。
2. 遍历：若 `count==0`，换候选为当前元素；若与候选相同 count++，否则 count--。
3. 题目保证多数存在，最终候选即为答案（无需二次验证）。

### 代码

```python
from typing import List


class Solution:
    def majorityElement(self, nums: List[int]) -> int:
        cand = count = 0
        for x in nums:
            if count == 0:
                cand = x
            count += 1 if x == cand else -1
        return cand
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(1) |

### 讲解

与 `solution.py` 一致。直觉：多数元素会「熬过」所有抵消。若未保证多数存在，需第二遍计数验证。

## 解法三：排序

### 思路

多数元素出现超过一半，排序后中间位置 `nums[n//2]` 一定是多数元素。

### 代码

```python
from typing import List


class Solution:
    def majorityElement(self, nums: List[int]) -> int:
        nums.sort()
        return nums[len(nums) // 2]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n log n) |
| 空间 | O(1) 或 O(log n) |

### 讲解

代码极简，但慢于投票法。适合作为 follow-up「不用额外空间但可改数组」的备选。

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 哈希计数 | O(n) | O(n) | 快速实现 |
| Boyer-Moore | O(n) | O(1) | 面试首选 |
| 排序 | O(n log n) | O(1) | 简洁备选 |

## 常见错误

1. **多数定义**：严格大于 n/2，不是 ≥。
2. **未验证候选**：169 题保证存在；229 题找超过 n/3 的元素需不同算法。
3. **count 初始化**：候选与 count 可同时置 0，见第一个元素时 count 变 1。
