# 寻找重复数（Find the Duplicate Number）

- 题目链接：https://leetcode.cn/problems/find-the-duplicate-number/
- 难度：中等
- 标签：位运算、数组、双指针、二分查找

## 题目描述

给定一个包含 `n + 1` 个整数的数组 `nums`，其数字都在 `[1, n]` 范围内（含 1 和 n），可知至少存在一个重复的整数。

假设 `nums` 只有**一个重复的整数**，返回这个重复的数。

你设计的解决方案必须**不修改**数组 `nums` 且只用常量级额外空间。

### 示例

```
输入：nums = [1,3,4,2,2]
输出：2

输入：nums = [3,1,3,4,2]
输出：3
```

### 约束

- `1 <= n <= 10^5`
- `nums.length == n + 1`
- `1 <= nums[i] <= n`
- `nums` 中只有一个整数出现两次或多次，其余整数只出现一次

---

## 解法一：Floyd 判圈（快慢指针，推荐，与 solution.py 一致）

### 思路

1. 把 `nums[i]` 看作从节点 `i` 指向节点 `nums[i]` 的边，形成链表。
2. 因存在重复值，必有环；环的入口即为重复数。
3. 阶段一：快慢指针相遇；阶段二：一指针从头、一指针从相遇点同步走，相遇处即重复数。

### 代码

```python
from typing import List


class Solution:
    def findDuplicate(self, nums: List[int]) -> int:
        slow = fast = nums[0]
        while True:
            slow = nums[slow]
            fast = nums[nums[fast]]
            if slow == fast:
                break
        slow = nums[0]
        while slow != fast:
            slow = nums[slow]
            fast = nums[fast]
        return slow
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(1) |

### 讲解

与 `solution.py` 一致。不修改数组、O(1) 空间，满足题面硬约束。索引 0 作起点即可，重复数即环入口。

## 解法二：二分答案

### 思路

1. 在 `[1, n]` 上二分候选值 `mid`。
2. 统计 `nums` 中 `≤ mid` 的个数 `cnt`；若 `cnt > mid`，说明重复数在左半，否则在右半。
3. 类似「抽屉原理」：值域 `[1,mid]` 最多 mid 个数，若实际更多则有重复。

### 代码

```python
from typing import List


class Solution:
    def findDuplicate(self, nums: List[int]) -> int:
        lo, hi = 1, len(nums) - 1
        while lo < hi:
            mid = (lo + hi) // 2
            cnt = sum(x <= mid for x in nums)
            if cnt > mid:
                hi = mid
            else:
                lo = mid + 1
        return lo
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n log n) |
| 空间 | O(1) |

### 讲解

不修改数组，思路与值域二分不同。面试可写，但 Floyd 更优。

## 解法三：哈希 / 排序（不满足约束）

### 思路

用 set 找重复，或排序后找相邻相等。违反「不修改数组」或「O(1) 空间」。

### 代码

```python
from typing import List


class Solution:
    def findDuplicate(self, nums: List[int]) -> int:
        seen = set()
        for x in nums:
            if x in seen:
                return x
            seen.add(x)
        return -1
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(n) |

### 讲解

仅作对比，说明题面约束的意义。实际面试先提哈希再优化到 Floyd。

---

## 解法对比

| 解法 | 时间 | 空间 | 修改数组 | 适用场景 |
|------|------|------|----------|----------|
| Floyd | O(n) | O(1) | 否 | 面试首选 |
| 二分值域 | O(n log n) | O(1) | 否 | 备选 |
| 哈希 | O(n) | O(n) | 否 | 不满足空间 |

## 常见错误

1. **第二阶段起点**：一个指针从 `nums[0]` 开始，不是从 0 下标。
2. **与 141 题混淆**：141 判断是否有环，本题求环入口（重复值）。
3. **修改数组**：用负数标记会破坏输入，题面禁止。
