# 下一个排列（Next Permutation）

- 题目链接：https://leetcode.cn/problems/next-permutation/
- 难度：中等
- 标签：数组、双指针

## 题目描述

整数数组的一个**排列** 就是将其所有成员以序列或线性顺序排列。

- 例如，`arr = [1,2,3]`，以下这些都可以视作 `arr` 的排列：`[1,2,3]`、`[1,3,2]`、`[3,1,2]`、`[2,3,1]` 等。

数组的**下一个排列** 是指字典序下一个更大的排列。若不存在下一个更大的排列，则将数字重新排列成最小的排列（即升序排列）。

- 例如，`arr = [1,2,3]` 的下一个排列是 `[1,3,2]`。
- 类似地，`arr = [2,3,1]` 的下一个排列是 `[3,1,2]`。
- 而 `arr = [3,2,1]` 没有下一个排列，需变为 `[1,2,3]`。

必须**原地**修改，只允许使用额外常数空间。

### 示例

```
输入：nums = [1,2,3]
输出：[1,3,2]

输入：nums = [3,2,1]
输出：[1,2,3]

输入：nums = [1,1,5]
输出：[1,5,1]
```

### 约束

- `1 <= nums.length <= 100`
- `0 <= nums[i] <= 100`

## 解法一：找下一个排列算法（推荐，与 solution.py 一致）

### 思路

1. **从右向左**找第一个 `nums[i] < nums[i+1]`，记为 `i`；若不存在则整个数组降序，直接反转变最小。
2. 从右向左找第一个 `nums[j] > nums[i]`，交换 `nums[i]` 与 `nums[j]`。
3. 反转 `nums[i+1:]` 使其为升序（交换后 suffix 仍是降序，反转即最小）。

### 代码

```python
from typing import List


class Solution:
    def nextPermutation(self, nums: List[int]) -> None:
        n = len(nums)
        i = n - 2
        while i >= 0 and nums[i] >= nums[i + 1]:
            i -= 1
        if i >= 0:
            j = n - 1
            while nums[j] <= nums[i]:
                j -= 1
            nums[i], nums[j] = nums[j], nums[i]
        nums[i + 1 :] = reversed(nums[i + 1 :])
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(1) |

### 讲解

与 `solution.py` 一致。例如 `[1,2,3]`：`i=1`，交换 2 与 3 得 `[1,3,2]`。`[3,2,1]` 无 `i`，直接反转。

## 解法二：暴力找下一个（理解用）

### 思路

1. 生成所有排列并排序，找当前排列的下一个。
2. n≤100 时不可行，仅说明题意。

### 代码

```python
from typing import List
import itertools


class Solution:
    def nextPermutation(self, nums: List[int]) -> None:
        perms = sorted(set(itertools.permutations(nums)))
        t = tuple(nums)
        idx = perms.index(t)
        nxt = perms[(idx + 1) % len(perms)]
        nums[:] = list(nxt)
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n! × n) |
| 空间 | O(n!) |

### 讲解

仅用于理解「字典序下一个」。面试严禁提交。

## 解法三：用栈找「可增位」（等价分析）

### 思路

与解法一相同，强调：要增大排列，尽量改靠右的数字；把 `i` 位换成右侧比它大的最小数字，后面再最小化。

### 代码

```python
from typing import List


class Solution:
    def nextPermutation(self, nums: List[int]) -> None:
        n = len(nums)
        k = -1
        for i in range(n - 2, -1, -1):
            if nums[i] < nums[i + 1]:
                k = i
                break
        if k == -1:
            nums.reverse()
            return
        for j in range(n - 1, k, -1):
            if nums[j] > nums[k]:
                nums[k], nums[j] = nums[j], nums[k]
                break
        left, right = k + 1, n - 1
        while left < right:
            nums[left], nums[right] = nums[right], nums[left]
            left += 1
            right -= 1
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(1) |

### 讲解

手写 reverse 替代切片，逻辑与解法一完全一致。注意 `i=-1` 时 `nums[i+1:]` 即整段反转。

## 常见错误

1. **找 j 时用 `<` 而非 `<=`**：需找严格大于 `nums[i]` 的。
2. **忘记反转 suffix**：交换后 suffix 仍是降序，必须变升序才是「下一个」。
3. **最大排列**：`i=-1` 时要反转整个数组，不是不动。
