# 两数之和（Two Sum）

- 题目链接：https://leetcode.cn/problems/two-sum/
- 难度：简单
- 标签：数组、哈希表

## 题目描述

给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出**和为目标值** `target` 的那**两个**整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。

### 示例

```
输入：nums = [2,7,11,15], target = 9
输出：[0,1]
解释：nums[0] + nums[1] == 9 ，返回 [0, 1]

输入：nums = [3,2,4], target = 6
输出：[1,2]

输入：nums = [3,3], target = 6
输出：[0,1]
```

### 约束

- `2 <= nums.length <= 10^4`
- `-10^9 <= nums[i] <= 10^9`
- `-10^9 <= target <= 10^9`
- **只会存在一个有效答案**

---

## 解法一：暴力枚举（最容易想到）

### 思路

用两层循环，枚举所有不同的下标对 `(i, j)`，检查 `nums[i] + nums[j]` 是否等于 `target`。

### 代码

```python
from typing import List


class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        n = len(nums)
        for i in range(n):
            for j in range(i + 1, n):
                if nums[i] + nums[j] == target:
                    return [i, j]
        return []
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n²) |
| 空间 | O(1) |

### 讲解

暴力法逻辑最直观：把所有可能的组合都试一遍。对于 Hot 100 第一题，很多初学者会先写这个版本。

**优点**：实现简单，不需要额外数据结构。  
**缺点**：当 `n` 接近 10⁴ 时，最坏情况约 10⁸ 次比较，容易超时。

---

## 解法二：哈希表（推荐，最优）

### 思路

核心问题：对于当前数 `nums[i]`，我们需要找的是 **`target - nums[i]`** 是否出现过。

如果不用哈希表，每次查找都要扫一遍前面的元素 → O(n)，总复杂度 O(n²)。  
如果用哈希表记录「值 → 下标」，查找变成 O(1) 平均时间。

**遍历过程：**

1. 当前下标 `i`，当前值 `num = nums[i]`
2. 计算 `need = target - num`
3. 若 `need` 已在哈希表 `seen` 中 → 返回 `[seen[need], i]`
4. 否则把 `num` 和下标 `i` 存入 `seen`，继续

> 注意：先查后存，避免同一个元素被用两次（例如 `[3, 3], target=6` 时不会误用同一个下标）。

### 代码

```python
from typing import List


class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        seen = {}  # value -> index
        for i, num in enumerate(nums):
            need = target - num
            if need in seen:
                return [seen[need], i]
            seen[num] = i
        return []
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n)，每个元素最多入表、查表各一次 |
| 空间 | O(n)，哈希表最坏存下全部元素 |

### 讲解

这是 LeetCode 最经典的「**空间换时间**」模板之一。

**为什么不会漏解？**  
当我们遍历到 `nums[i]` 时，哈希表里存的是 `i` 之前所有元素。若存在某个 `j < i` 使得 `nums[j] + nums[i] = target`，则在处理 `i` 时一定能从表中找到 `nums[j]`。

**为什么先查后存？**  
若先存再查，对于 `nums = [3], target = 6` 这类边界（虽然本题保证有解且至少两个元素），或 `[3, 3]` 若先存 `3` 再查 `need=3`，可能错误地用到同一位置。先查后存保证配对的是**两个不同下标**。

---

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 暴力枚举 | O(n²) | O(1) | 理解题意、小数据 |
| 哈希表 | O(n) | O(n) | 面试与提交首选 |

---

## 常见错误

1. **返回数值而不是下标**：题目要的是 index，不是 value。
2. **重复用同一元素**：`[3,3], target=6` 应返回 `[0,1]`，不能 `[0,0]`。
3. **哈希表存错内容**：应存「值 → 下标」，查找时用 `target - num` 作为 key。
