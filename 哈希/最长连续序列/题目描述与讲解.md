# 最长连续序列（Longest Consecutive Sequence）

- 题目链接：https://leetcode.cn/problems/longest-consecutive-sequence/
- 难度：中等
- 标签：并查集、数组、哈希表

## 题目描述

给定一个未排序的整数数组 `nums`，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 **O(n)** 的算法解决此问题。

### 示例

```
输入：nums = [100, 4, 200, 1, 3, 2]
输出：4
解释：最长连续序列是 [1, 2, 3, 4]，长度为 4。

输入：nums = [0, 3, 7, 2, 5, 8, 4, 6, 0, 1]
输出：9
```

### 约束

- `0 <= nums.length <= 10^5`
- `-10^9 <= nums[i] <= 10^9`

---

## 解法一：排序后线性扫描

### 思路

先排序，再遍历数组统计连续段长度。注意跳过重复元素。

### 代码

```python
from typing import List


class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        if not nums:
            return 0
        nums.sort()
        ans = 1
        cur = 1
        for i in range(1, len(nums)):
            if nums[i] == nums[i - 1]:
                continue
            if nums[i] == nums[i - 1] + 1:
                cur += 1
            else:
                cur = 1
            ans = max(ans, cur)
        return ans
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n log n)，排序主导 |
| 空间 | O(1) 或 O(log n)（排序栈空间，视语言实现） |

### 讲解

思路直观，但题目明确要求 O(n)，排序不满足要求。可作为理解题的入门解法。

## 解法二：哈希集合 + 只从序列起点扩展（推荐，最优）

### 思路

1. 把所有数放入哈希集合 `num_set`，O(1) 查找。
2. 对每个 `num`，**仅当 `num - 1` 不在集合中**时，说明 `num` 是某条连续序列的**起点**。
3. 从起点不断检查 `num + 1`、`num + 2`... 是否在集合中，统计长度。

### 代码

```python
from typing import List


class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        num_set = set(nums)
        ans = 0

        for num in num_set:
            if num - 1 not in num_set:
                cur = num
                length = 1
                while cur + 1 in num_set:
                    cur += 1
                    length += 1
                ans = max(ans, length)

        return ans
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n)，每个数最多被「作为起点扩展」访问一次，被扩展时最多访问常数次 |
| 空间 | O(n)，哈希集合 |

### 讲解

**为什么是 O(n)？**  
看似有嵌套 while，但每个数最多进入内层循环一次（作为某条连续序列的一部分被数过）。所有 while 的总迭代次数 ≤ n。

**为什么要判断 `num - 1 not in num_set`？**  
避免从序列中间重复扩展。例如序列 `1,2,3,4`：

- 只在 `num=1` 时扩展，得到长度 4
- 若从 `2,3,4` 也扩展，会做大量重复工作

**示例 walkthrough（nums = [100, 4, 200, 1, 3, 2]）：**

```
num_set = {100, 4, 200, 1, 3, 2}

100: 99 不在集合 → 只含 100，长度 1
4:   3 在集合 → 跳过（不是起点）
200: 199 不在 → 长度 1
1:   0 不在 → 扩展 1→2→3→4，长度 4  ← 答案
```

## 解法三：并查集（了解即可）

### 思路

对相邻数字（差值为 1）做 union，最后找最大连通分量大小。

### 代码

```python
from typing import List


class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        if not nums:
            return 0

        parent = {}
        size = {}

        def find(x: int) -> int:
            if parent[x] != x:
                parent[x] = find(parent[x])
            return parent[x]

        for num in set(nums):
            parent[num] = num
            size[num] = 1

        for num in parent:
            if num + 1 in parent:
                ra, rb = find(num), find(num + 1)
                if ra != rb:
                    parent[rb] = ra
                    size[ra] += size[rb]

        return max(size.values())
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | 接近 O(n × α(n))，但需先知道哪些数相邻，实践上不如哈希法简洁 |
| 空间 | O(n) |

### 讲解

并查集可行，但本题用哈希集合更直接，面试优先讲解法二。

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 排序扫描 | O(n log n) | O(1) | 理解题意 |
| 哈希 + 起点扩展 | O(n) | O(n) | 面试与提交首选 |
| 并查集 | O(n α(n)) | O(n) | 扩展知识 |

---

## 常见错误

1. **每个数都向内扩展**：不判断起点，导致 O(n²) 超时。
2. **忽略重复元素**：如 `[1,2,2,3]`，排序法需跳过重复；哈希法用 set 天然去重。
3. **空数组返回 1 而不是 0**：`nums = []` 应返回 0。
4. **用排序却声称 O(n)**：面试时要能说清复杂度。
