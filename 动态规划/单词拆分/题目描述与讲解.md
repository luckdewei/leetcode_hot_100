# 单词拆分（Word Break）

- 题目链接：https://leetcode.cn/problems/word-break/
- 难度：中等
- 标签：字典树、记忆化搜索、数组、哈希表、字符串、动态规划

## 题目描述

给定一个**非空**字符串 `s` 和一个包含**非空**单词列表的字典 `wordDict`，判断 `s` 是否可以被空格拆分为一个或多个在字典中出现的单词。

**说明**：拆分时可以重复使用字典中的单词。你可以假设字典中没有重复的单词。

### 示例

```
输入：s = "leetcode", wordDict = ["leet","code"]
输出：true
解释：可以拆成 "leet" + "code"

输入：s = "applepenapple", wordDict = ["apple","pen"]
输出：true
解释：apple + pen + apple

输入：s = "catsandog", wordDict = ["cats","dog","sand","and","cat"]
输出：false
```

### 约束

- `1 <= s.length <= 300`
- `1 <= wordDict.length <= 1000`
- `1 <= wordDict[i].length <= 20`
- `s` 和 `wordDict[i]` 仅由小写英文字母组成
- `wordDict` 中所有字符串互不相同

## 解法一：记忆化 DFS

### 思路

1. 定义 `dfs(start)`：子串 `s[start:]` 能否被拆分。
2. 枚举以 `start` 开头的所有前缀，若在字典中且 `dfs(start+len(word))` 为真，则返回真。
3. 用 memo 缓存每个 start 的结果。

### 代码

```python
from typing import List


class Solution:
    def wordBreak(self, s: str, wordDict: List[str]) -> bool:
        words = set(wordDict)
        memo = {}

        def dfs(start: int) -> bool:
            if start == len(s):
                return True
            if start in memo:
                return memo[start]
            for end in range(start + 1, len(s) + 1):
                if s[start:end] in words and dfs(end):
                    memo[start] = True
                    return True
            memo[start] = False
            return False

        return dfs(0)
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n² × m)，n 为串长，m 为平均单词长（哈希查找 O(m)） |
| 空间 | O(n) |

### 讲解

自顶向下直观：尝试第一个单词选什么，再递归后面。剪枝靠 memo，避免重复子问题。

## 解法二：一维 DP（推荐，与 solution.py 一致）

### 思路

1. `dp[i]`：`s[0:i]`（前 i 个字符）能否被拆分。
2. `dp[0] = True`（空串）。
3. 对每个 `i`，枚举分割点 `j`：若 `dp[j]` 为真且 `s[j:i]` 在字典中，则 `dp[i]=True`。

### 代码

```python
from typing import List


class Solution:
    def wordBreak(self, s: str, wordDict: List[str]) -> bool:
        words = set(wordDict)
        n = len(s)
        dp = [False] * (n + 1)
        dp[0] = True
        for i in range(1, n + 1):
            for j in range(i):
                if dp[j] and s[j:i] in words:
                    dp[i] = True
                    break
        return dp[n]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n² × m) |
| 空间 | O(n) |

### 讲解

与 `solution.py` 一致。`dp[i]` 表示前缀可拆分，转移是「找一个合法的最后一段单词」。找到一种即可 break。

## 解法三：BFS 按长度扩展

### 思路

从位置 0 出发，若 `s[0:j]` 在字典中则 j 入队；从队列取出位置 i，再尝试延伸……能到达 n 则 true。

### 代码

```python
from collections import deque
from typing import List


class Solution:
    def wordBreak(self, s: str, wordDict: List[str]) -> bool:
        words = set(wordDict)
        n = len(s)
        queue = deque([0])
        visited = [False] * n
        while queue:
            start = queue.popleft()
            if visited[start]:
                continue
            visited[start] = True
            for end in range(start + 1, n + 1):
                if s[start:end] in words:
                    if end == n:
                        return True
                    queue.append(end)
        return False
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n² × m) |
| 空间 | O(n) |

### 讲解

把「可到达的下标」看作图上的节点，单词匹配是边。与 DP 本质相同，换了一种表述。

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 记忆化 DFS | O(n²m) | O(n) | 递归思路 |
| 一维 DP | O(n²m) | O(n) | 面试首选 |
| BFS | O(n²m) | O(n) | 图论视角 |

## 常见错误

1. **dp 下标**：`dp[i]` 对应前 i 个字符，`dp[0]=True`。
2. **子串边界**：`s[j:i]` 左闭右开，不是 `s[j:i+1]`。
3. **与 140 题混淆**：140 题要输出所有拆分方案，本题只要判断能否。
