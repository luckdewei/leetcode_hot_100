# 零钱兑换（Coin Change）

- 题目链接：https://leetcode.cn/problems/coin-change/
- 难度：中等
- 标签：广度优先搜索、数组、动态规划

## 题目描述

给你一个整数数组 `coins` 表示不同面额的硬币，以及一个整数 `amount` 表示总金额。

计算并返回可以凑成总金额所需的**最少硬币个数**。如果没有任何一种硬币组合能组成总金额，返回 **-1**。

你可以认为每种硬币的数量是无限的。

### 示例

```
输入：coins = [1,2,5], amount = 11
输出：3
解释：11 = 5 + 5 + 1

输入：coins = [2], amount = 3
输出：-1

输入：coins = [1], amount = 0
输出：0
```

### 约束

- `1 <= coins.length <= 12`
- `1 <= coins[i] <= 2^31 - 1`
- `0 <= amount <= 10^4`

---

## 解法一：BFS 最短路径

### 思路

1. 从 amount 倒推：每次减去一种硬币面额，求最少步数到 0。
2. 或从 0 出发每次加硬币，BFS 层数即硬币个数。
3. 用 visited 避免重复状态。

### 代码

```python
from collections import deque
from typing import List


class Solution:
    def coinChange(self, coins: List[int], amount: int) -> int:
        if amount == 0:
            return 0
        queue = deque([0])
        visited = {0}
        step = 0
        while queue:
            step += 1
            for _ in range(len(queue)):
                cur = queue.popleft()
                for c in coins:
                    nxt = cur + c
                    if nxt == amount:
                        return step
                    if nxt < amount and nxt not in visited:
                        visited.add(nxt)
                        queue.append(nxt)
        return -1
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(amount × len(coins)) |
| 空间 | O(amount) |

### 讲解

BFS 天然求「最少步数」。与 DP 等价，但常数较大，面试更推荐 DP。

## 解法二：完全背包 DP（推荐，与 solution.py 一致）

### 思路

1. `dp[i]`：凑成金额 `i` 所需最少硬币数。
2. 对每个金额 `i`，尝试每种硬币 `c`：`dp[i] = min(dp[i], dp[i-c] + 1)`。
3. 初始 `dp[0]=0`，其余设为 `amount+1` 表示不可达；最终若 `dp[amount] > amount` 返回 -1。

### 代码

```python
from typing import List


class Solution:
    def coinChange(self, coins: List[int], amount: int) -> int:
        dp = [0] + [amount + 1] * amount
        for i in range(1, amount + 1):
            for c in coins:
                if c <= i:
                    dp[i] = min(dp[i], dp[i - c] + 1)
        return dp[amount] if dp[amount] <= amount else -1
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(amount × len(coins)) |
| 空间 | O(amount) |

### 讲解

与 `solution.py` 一致。外层遍历金额、内层遍历硬币是「完全背包最少个数」标准写法。不可达用 `amount+1` 哨兵，比 inf 更直观。

## 解法三：记忆化 DFS（自顶向下）

### 思路

定义 `dfs(rem)` 为凑成 `rem` 的最少硬币数，枚举最后一枚硬币是什么面额。

### 代码

```python
from typing import List


class Solution:
    def coinChange(self, coins: List[int], amount: int) -> int:
        memo = {}

        def dfs(rem: int) -> int:
            if rem == 0:
                return 0
            if rem < 0:
                return amount + 1
            if rem in memo:
                return memo[rem]
            memo[rem] = min(dfs(rem - c) for c in coins) + 1
            return memo[rem]

        ans = dfs(amount)
        return ans if ans <= amount else -1
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(amount × len(coins)) |
| 空间 | O(amount) |

### 讲解

自顶向下与自底向上 DP 等价，适合从递归思路过渡。注意 `min` 空集与边界。

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| BFS | O(A×C) | O(A) | 理解最短路径 |
| 完全背包 DP | O(A×C) | O(A) | 面试首选 |
| 记忆化 DFS | O(A×C) | O(A) | 递归爱好者 |

*A=amount，C=len(coins)*

---

## 常见错误

1. **0-1 背包顺序**：完全背包应外层金额、内层硬币；若内层金额需倒序（本题无限枚无此问题）。
2. **不可达判断**：`dp[amount] == amount+1` 时返回 -1，不是 0。
3. **与 518 题混淆**：518 求组合数，本题求最少个数。
