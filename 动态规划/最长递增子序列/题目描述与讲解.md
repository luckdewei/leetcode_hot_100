# 最长递增子序列（Longest Increasing Subsequence）

- 题目链接：https://leetcode.cn/problems/longest-increasing-subsequence/
- 难度：中等
- 标签：数组、二分查找、动态规划

## 题目描述

给你一个整数数组 `nums`，找到其中**最长严格递增子序列**的长度。

**子序列** 是由数组派生而来的序列，删除（或不删除）数组中的元素而不改变其余元素的顺序。例如，`[3,6,2,7]` 是 `[0,3,1,6,2,2,7]` 的子序列。

### 示例

```
输入：nums = [10,9,2,5,3,7,101,18]
输出：4
解释：最长递增子序列是 [2,3,7,101]，长度为 4

输入：nums = [0,1,0,3,2,3]
输出：4

输入：nums = [7,7,7,7,7,7,7]
输出：1
```

### 约束

- `1 <= nums.length <= 2500`
- `-10^4 <= nums[i] <= 10^4`

---

## 解法一：O(n²) DP

### 思路

1. `dp[i]`：以 `nums[i]` **结尾**的最长递增子序列长度。
2. 对每个 `i`，看所有 `j < i` 且 `nums[j] < nums[i]`：`dp[i] = max(dp[i], dp[j] + 1)`。
3. 答案为 `max(dp)`，初始均为 1。

### 代码

```python
from typing import List


class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        n = len(nums)
        dp = [1] * n
        for i in range(n):
            for j in range(i):
                if nums[j] < nums[i]:
                    dp[i] = max(dp[i], dp[j] + 1)
        return max(dp)
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n²) |
| 空间 | O(n) |

### 讲解

最经典的 LIS DP，n=2500 时约 600 万次运算，可能超时。适合理解「以 i 结尾」的状态定义。

## 解法二：贪心 + 二分（推荐，与 solution.py 一致）

### 思路

1. 维护数组 `tails`，`tails[k]` 表示长度为 `k+1` 的递增子序列的**最小末尾元素**。
2. 遍历 `x`：在 `tails` 中二分找第一个 `≥ x` 的位置，替换或追加。
3. `tails` 长度即为 LIS 长度。

### 代码

```python
from typing import List
import bisect


class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        tails = []
        for x in nums:
            pos = bisect.bisect_left(tails, x)
            if pos == len(tails):
                tails.append(x)
            else:
                tails[pos] = x
        return len(tails)
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n log n) |
| 空间 | O(n) |

### 讲解

与 `solution.py` 一致。贪心直觉：同样长度下末尾越小，后面越容易接更长序列。`bisect_left` 保证严格递增（相等时替换，不延长）。

## 解法三：耐心排序 + 二分（等价写法）

### 思路

把 `tails` 想象成若干堆牌，每张牌只能放在第一比它大的牌上；堆数即 LIS 长度。实现与解法二相同，用线性扫描代替 bisect 仅供理解。

### 代码

```python
from typing import List


class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        piles = []
        for x in nums:
            lo, hi = 0, len(piles)
            while lo < hi:
                mid = (lo + hi) // 2
                if piles[mid] < x:
                    lo = mid + 1
                else:
                    hi = mid
            if lo == len(piles):
                piles.append(x)
            else:
                piles[lo] = x
        return len(piles)
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n log n) |
| 空间 | O(n) |

### 讲解

手写二分与 `bisect_left` 等价。面试中可直接用 bisect，更简洁。

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| O(n²) DP | O(n²) | O(n) | 小数据、需具体序列 |
| 贪心+二分 | O(n log n) | O(n) | 面试首选 |
| 耐心排序 | O(n log n) | O(n) | 理解贪心正确性 |

## 常见错误

1. **严格递增**：相等不能接在后面，用 `bisect_left` 不是 `bisect_right`（非严格递增才用 right）。
2. **dp 答案**：O(n²) 写法要 `max(dp)`，不是 `dp[-1]`。
3. **与 674 题混淆**：674 求最长连续递增子数组，不是子序列。
