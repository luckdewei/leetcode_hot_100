# 找到字符串中所有字母异位词（Find All Anagrams in a String）

- 题目链接：https://leetcode.cn/problems/find-all-anagrams-in-a-string/
- 难度：中等
- 标签：哈希表、字符串、滑动窗口

## 题目描述

给定两个字符串 `s` 和 `p`，找到 `s` 中所有 `p` 的**异位词**的子串，返回这些子串的起始索引。不考虑答案输出的顺序。

**异位词**指由相同字母重排列形成的字符串（包括相同的字符串）。

### 示例

```
输入：s = "cbaebabacd", p = "abc"
输出：[0,6]
解释：
起始索引 0 的子串是 "cba"，是 "abc" 的异位词。
起始索引 6 的子串是 "bac"，是 "abc" 的异位词。

输入：s = "abab", p = "ab"
输出：[0,1,2]
解释：
起始索引 0 的子串是 "ab"，是 "ab" 的异位词。
起始索引 1 的子串是 "ba"，是 "ab" 的异位词。
起始索引 2 的子串是 "ab"，是 "ab" 的异位词。
```

### 约束

- `1 <= s.length, p.length <= 3 * 10^4`
- `s` 和 `p` 仅包含小写英文字母

---

## 解法一：排序比较每个子串

### 思路

枚举 `s` 中每个长度为 `len(p)` 的子串，排序后与 `sorted(p)` 比较。

### 代码

```python
from typing import List


class Solution:
    def findAnagrams(self, s: str, p: str) -> List[int]:
        k = len(p)
        if k > len(s):
            return []
        target = sorted(p)
        ans = []
        for i in range(len(s) - k + 1):
            if sorted(s[i : i + k]) == target:
                ans.append(i)
        return ans
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n × k log k) |
| 空间 | O(k) |

### 讲解

思路直接，但 n、k 最大 3×10⁴ 时会超时。适合理解「异位词 = 排序相同」。

## 解法二：定长滑动窗口 + 频次数组（推荐，最优）

### 思路

1. 统计 `p` 的字符频次 `need`。
2. 维护长度为 `k = len(p)` 的窗口频次 `window`。
3. `right` 右扩时加入字符；当窗口长度达到 `k` 时，若 `window == need` 则记录起点 `left = right - k + 1`。
4. 然后移除 `s[left]`，窗口左移一格，继续。

### 代码

```python
from collections import Counter
from typing import List


class Solution:
    def findAnagrams(self, s: str, p: str) -> List[int]:
        if len(p) > len(s):
            return []

        need = Counter(p)
        window = Counter()
        ans = []
        k = len(p)

        for right, ch in enumerate(s):
            window[ch] += 1

            left = right - k + 1
            if left < 0:
                continue

            if window == need:
                ans.append(left)

            window[s[left]] -= 1
            if window[s[left]] == 0:
                del window[s[left]]

        return ans
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n)，每个字符最多进出窗口一次 |
| 空间 | O(1)，字符集仅 26 个小写字母 |

### 讲解

**定长窗口模板：**

```
for right in range(n):
    加入 s[right]
    if 窗口长度 == k:
        检查是否匹配
        移除 s[right - k + 1]
```

**为何用 Counter 比较？**  
26 个字母的频次向量，相等即异位词。也可维护 `matched` 变量，用 O(1) 判断是否匹配，避免每次比较整个 Counter。

**示例（s = "cbaebabacd", p = "abc"）：**

```
窗口 "cba" @0 → 频次与 abc 相同 → 记录 0
...
窗口 "bac" @6 → 匹配 → 记录 6
```

## 解法三：滑动窗口 + 匹配计数（常数优化）

### 思路

用长度 26 的数组代替 Counter，维护当前有多少个字母的频次已匹配 `need`，当 `matched == need_letters` 时记录答案。

### 代码

```python
from typing import List


class Solution:
    def findAnagrams(self, s: str, p: str) -> List[int]:
        if len(p) > len(s):
            return []

        need = [0] * 26
        window = [0] * 26
        for ch in p:
            need[ord(ch) - ord("a")] += 1

        need_letters = sum(1 for x in need if x > 0)
        matched = 0
        ans = []
        k = len(p)

        for right, ch in enumerate(s):
            idx = ord(ch) - ord("a")
            if window[idx] == need[idx]:
                matched -= 1
            window[idx] += 1
            if window[idx] == need[idx]:
                matched += 1

            left = right - k + 1
            if left < 0:
                continue

            if matched == need_letters:
                ans.append(left)

            left_idx = ord(s[left]) - ord("a")
            if window[left_idx] == need[left_idx]:
                matched -= 1
            window[left_idx] -= 1
            if window[left_idx] == need[left_idx]:
                matched += 1

        return ans
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(1) |

### 讲解

面试写 Counter 版足够；优化版避免每次 O(26) 比较，是工业级写法。

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 排序比较 | O(n × k log k) | O(k) | 理解题意 |
| 定长窗口 + Counter | O(n) | O(1) | 面试首选 |
| 匹配计数优化 | O(n) | O(1) | 追求极致常数 |

## 常见错误

1. **窗口长度不对**：必须是 `len(p)`，不是任意长。
2. **忘记左移时减少频次**：会导致窗口频次漂移。
3. **p 比 s 长**：应直接返回 `[]`。
4. **与「字母异位词分组」混淆**：本题是在长串中找子串位置。
