# 二叉树中的最大路径和（Binary Tree Maximum Path Sum）

- 题目链接：https://leetcode.cn/problems/binary-tree-maximum-path-sum/
- 难度：困难
- 标签：树、深度优先搜索、动态规划、二叉树

## 题目描述

二叉树中的**路径**是从树中任意节点出发，沿父节点-子节点连接，达到任意节点的序列。同一节点在一条路径中最多出现一次。

给你一个二叉树的根节点 `root`，返回**最大路径和**（路径上节点值之和）。

### 示例

```
输入：root = [1,2,3]
输出：6
解释：最优路径 2 → 1 → 3，和为 6

输入：root = [-10,9,20,null,null,15,7]
输出：42
解释：最优路径 15 → 20 → 7，和为 42

输入：root = [-3]
输出：-3
```

### 约束

- 树中节点数目范围 `[1, 3 * 10^4]`
- `-1000 <= Node.val <= 1000`

---

## 解法一：后序 DFS + 全局最大值（推荐，同 solution.py）

### 思路

对每个节点计算「向下延伸的最大贡献」：左、右子树贡献若为负则舍弃（取 max(0)）。经过该节点的「弧形路径」和为 `node.val + left + right`，用全局变量更新答案。返回给父节点的是 `node.val + max(left, right)`（只能选一侧向上）。

### 代码

```python
from typing import Optional


class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def maxPathSum(self, root: Optional[TreeNode]) -> int:
        self.ans = float("-inf")

        def gain(node: Optional[TreeNode]) -> int:
            if not node:
                return 0
            left = max(gain(node.left), 0)
            right = max(gain(node.right), 0)
            self.ans = max(self.ans, node.val + left + right)
            return node.val + max(left, right)

        gain(root)
        return self.ans
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(h) |

### 讲解

**两个概念分离**：`ans` 记录可拐弯的完整路径；返回值是单链向上延伸，供父节点使用。负贡献子树必须丢弃，否则拖累和。

## 解法二：显式后序 + 元组返回

### 思路

返回 `(向下最大链, 包含当前的最大路径)`，逻辑与解法一等价，结构更显式。

### 代码

```python
class Solution:
    def maxPathSum(self, root: Optional[TreeNode]) -> int:
        def dfs(node: Optional[TreeNode]):
            if not node:
                return 0, float("-inf")
            ll, lp = dfs(node.left)
            rl, rp = dfs(node.right)
            down = node.val + max(ll, rl, 0)
            through = node.val + max(ll, 0) + max(rl, 0)
            best = max(lp, rp, through, node.val)
            return down, best

        return dfs(root)[1]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(h) |

### 讲解

`best` 还需与 `node.val` 比较，处理全负树时单节点为最优的情况。

## 解法三：暴力枚举路径（理解用）

### 思路

枚举每个节点为路径最高点，计算左链最大 + 右链最大 + 根值。链最大需递归求（允许负值时的单链最大）。

### 代码

```python
class Solution:
    def maxPathSum(self, root: Optional[TreeNode]) -> int:
        def max_gain(node: Optional[TreeNode]) -> int:
            if not node:
                return 0
            left = max(max_gain(node.left), 0)
            right = max(max_gain(node.right), 0)
            return node.val + max(left, right)

        def walk(node: Optional[TreeNode]) -> int:
            if not node:
                return float("-inf")
            left = max(max_gain(node.left), 0)
            right = max(max_gain(node.right), 0)
            best = max(
                node.val + left + right,
                walk(node.left),
                walk(node.right),
            )
            return best

        return walk(root)
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n²) 最坏重复计算 |
| 空间 | O(h) |

### 讲解

可配合记忆化达到 O(n)，本质仍是一次后序。暴力版帮助理解「枚举拐弯点」。

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 后序全局 max | O(n) | O(h) | 面试首选 |
| 元组返回 | O(n) | O(h) | 逻辑拆分清晰 |
| 暴力枚举 | O(n²) | O(h) | 理解拐弯点 |

## 常见错误

1. **返回值用左右之和**：向上只能选一侧，不能同时把左右给父节点。
2. **忘记处理负数**：`max(gain, 0)` 丢弃负子树；但单节点负值仍可能是全局 max。
3. **ans 初值 0**：全负树时应用 `-inf` 初始化。
