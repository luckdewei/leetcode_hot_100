# 二叉搜索树中第 K 小的元素（Kth Smallest Element in a BST）

- 题目链接：https://leetcode.cn/problems/kth-smallest-element-in-a-bst/
- 难度：中等
- 标签：树、深度优先搜索、二叉搜索树、二叉树

## 题目描述

给定一个二叉搜索树的根节点 `root` 和整数 `k`，返回其中第 **`k` 小**的元素（1 索引）。

### 示例

```
输入：root = [3,1,4,null,2], k = 1
输出：1

输入：root = [5,3,6,2,4,null,null,1], k = 3
输出：3
```

### 约束

- 树中节点数 `n` 满足 `1 <= k <= n <= 10^4`
- `0 <= Node.val <= 10^4`

---

## 解法一：迭代中序（推荐，同 solution.py）

### 思路

BST 中序为升序。迭代中序，每弹出一个节点 `k -= 1`，为 0 时即为第 k 小。

### 代码

```python
from typing import Optional


class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def kthSmallest(self, root: Optional[TreeNode], k: int) -> int:
        stack, cur = [], root
        while cur or stack:
            while cur:
                stack.append(cur)
                cur = cur.left
            cur = stack.pop()
            k -= 1
            if k == 0:
                return cur.val
            cur = cur.right
        return -1
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(h + k)，k 小时可提前终止 |
| 空间 | O(h) |

### 讲解

相比完整中序 O(n)，提前停止更优。面试首选写法。

## 解法二：递归中序

### 思路

递归左-访问-右，用计数器记录当前是第几个。

### 代码

```python
class Solution:
    def kthSmallest(self, root: Optional[TreeNode], k: int) -> int:
        self.count = 0
        self.ans = 0

        def dfs(node: Optional[TreeNode]) -> None:
            if not node:
                return
            dfs(node.left)
            self.count += 1
            if self.count == k:
                self.ans = node.val
                return
            dfs(node.right)

        dfs(root)
        return self.ans
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(h + k) |
| 空间 | O(h) |

### 讲解

逻辑与迭代相同，找到后可剪枝（本实现仍会继续遍历，可加 flag 优化）。

## 解法三：记录子树大小（进阶）

### 思路

预处理每个节点左子树大小 `size[left]`。若 `k <= size[left]` 进左子树；若 `k == size[left]+1` 为当前根；否则 `k -= size[left]+1` 进右子树。需可修改树或父指针。

### 代码

```python
class Solution:
    def kthSmallest(self, root: Optional[TreeNode], k: int) -> int:
        def size(node: Optional[TreeNode]) -> int:
            if not node:
                return 0
            return 1 + size(node.left) + size(node.right)

        while root:
            left_size = size(root.left)
            if k <= left_size:
                root = root.left
            elif k == left_size + 1:
                return root.val
            else:
                k -= left_size + 1
                root = root.right
        return -1
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) 每次算 size；可 O(1) 均摊若维护 size |
| 空间 | O(h) |

### 讲解

LeetCode 进阶要求多次查询时，应在节点上缓存 `left_count` 并在增删时更新。

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 迭代中序 | O(h+k) | O(h) | 单次查询，面试首选 |
| 递归中序 | O(h+k) | O(h) | 代码短 |
| 子树大小 | O(log n)* | O(1) | 多次查询（需维护） |

---

## 常见错误

1. **用前序/后序**：BST 第 k 小必须中序。
2. **k 从 0 开始**：题目为 1 索引，第一次弹出应为 k=1。
3. **非 BST 套用**：普通二叉树需完整中序排序，O(n log n)。
