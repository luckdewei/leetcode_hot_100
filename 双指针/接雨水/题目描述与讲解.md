# 接雨水（Trapping Rain Water）

- 题目链接：https://leetcode.cn/problems/trapping-rain-water/
- 难度：困难
- 标签：栈、数组、双指针、动态规划

## 题目描述

给定 `n` 个非负整数表示每个宽度为 `1` 的柱子的高度图，计算按此排列的柱子，下雨之后能接多少雨水。

### 示例

```
输入：height = [0,1,0,2,1,0,1,3,2,1,2,1]
输出：6
解释：上面是由数组 [0,1,0,2,1,0,1,3,2,1,2,1] 表示的高度图，
     在这种情况下，可以接 6 个单位的雨水。

输入：height = [4,2,0,3,2,5]
输出：9
```

### 约束

- `n == height.length`
- `1 <= n <= 2 * 10^4`
- `0 <= height[i] <= 10^5`

---

## 问题建模

位置 `i` 能接的雨水高度为：

```
water(i) = min(左侧最高, 右侧最高) - height[i]
```

若结果为负则取 0（柱子比水位高）。总雨水为所有位置之和。

---

## 解法一：按列累加 + 预处理左右最大值

### 思路

先预处理 `left_max[i]`、`right_max[i]`，再对每个位置计算接水量。

### 代码

```python
from typing import List


class Solution:
    def trap(self, height: List[int]) -> int:
        n = len(height)
        if n == 0:
            return 0

        left_max = [0] * n
        right_max = [0] * n
        left_max[0] = height[0]
        for i in range(1, n):
            left_max[i] = max(left_max[i - 1], height[i])

        right_max[n - 1] = height[n - 1]
        for i in range(n - 2, -1, -1):
            right_max[i] = max(right_max[i + 1], height[i])

        water = 0
        for i in range(n):
            water += min(left_max[i], right_max[i]) - height[i]
        return water
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(n) |

### 讲解

直观好写，空间可优化为 O(1)（见双指针法）。

## 解法二：双指针（推荐，最优）

### 思路

`left`、`right` 双指针，`left_max`、`right_max` 记录两侧当前最高柱。

- 若 `height[left] < height[right]`：说明**右侧一定有足够高的墙**（因为 `right_max >= height[right] > height[left]`），此时按 `left_max` 计算 `left` 位置接水，然后 `left++`。
- 否则处理 `right` 侧，对称。

### 代码

```python
from typing import List


class Solution:
    def trap(self, height: List[int]) -> int:
        if not height:
            return 0

        left, right = 0, len(height) - 1
        left_max, right_max = 0, 0
        water = 0

        while left < right:
            if height[left] < height[right]:
                left_max = max(left_max, height[left])
                water += left_max - height[left]
                left += 1
            else:
                right_max = max(right_max, height[right])
                water += right_max - height[right]
                right -= 1

        return water
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(1) |

### 讲解

**为什么矮侧可以先算？**  
当 `height[left] < height[right]` 时，水位由矮侧左边的 `left_max` 和右侧至少为 `height[right]` 的墙共同决定，有效水位为 `min(left_max, 右侧某高度) = left_max`（因为 left_max ≤ height[left] < height[right]）。

因此 `left` 位置接水量只取决于 `left_max`，可安全结算。

**示例（height = [4, 2, 0, 3, 2, 5]）：**

双指针从两端向中间，累计得总水量 **9**。

## 解法三：单调栈

### 思路

维护递减单调栈（存下标）。当当前柱高于栈顶，说明形成「凹槽」，按层或按宽计算积水并弹出。

### 代码

```python
from typing import List


class Solution:
    def trap(self, height: List[int]) -> int:
        stack = []
        water = 0
        for i, h in enumerate(height):
            while stack and h > height[stack[-1]]:
                bottom = stack.pop()
                if not stack:
                    break
                width = i - stack[-1] - 1
                bounded_height = min(h, height[stack[-1]]) - height[bottom]
                water += width * bounded_height
            stack.append(i)
        return water
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(n) |

### 讲解

单调栈适合「两侧都有更高柱子形成凹槽」的场景，是接雨水经典解法之一。与双指针相比代码稍复杂，但有助于理解栈题。

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 预处理左右最大 | O(n) | O(n) | 最易理解 |
| 双指针 | O(n) | O(1) | 面试首选 |
| 单调栈 | O(n) | O(n) | 栈专题 |

---

## 常见错误

1. **与盛水容器混淆**：盛水是选两条线围一个矩形；接水是每列单独算，再求和。
2. **直接用全局 max 当水位**：每列要看左右两侧最大值的最小值。
3. **双指针移动长侧**：应处理当前较矮的一侧。
4. **忘记累加**：结果是所有位置接水量之和，不是最大单格。
