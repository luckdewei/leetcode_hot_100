# 移动零（Move Zeroes）

- 题目链接：https://leetcode.cn/problems/move-zeroes/
- 难度：简单
- 标签：数组、双指针

## 题目描述

给定一个数组 `nums`，编写一个函数将所有 `0` 移动到数组的末尾，同时保持非零元素的相对顺序。

**请注意**，必须在不复制数组的情况下原地对数组进行操作。

### 示例

```
输入：nums = [0,1,0,3,12]
输出：[1,3,12,0,0]

输入：nums = [0]
输出：[0]
```

### 约束

- `1 <= nums.length <= 10^4`
- `-2^31 <= nums[i] <= 2^31 - 1`

---

## 解法一：两次遍历（先收集非零，再补零）

### 思路

第一遍把非零元素按顺序写到数组前部；第二遍把剩余位置填 0。

### 代码

```python
from typing import List


class Solution:
    def moveZeroes(self, nums: List[int]) -> None:
        write = 0
        for num in nums:
            if num != 0:
                nums[write] = num
                write += 1
        while write < len(nums):
            nums[write] = 0
            write += 1
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(1) |

### 讲解

逻辑清晰，但非零元素会被**赋值两次**（先写到前部，若原本就在前部则有一次冗余写）。数据量大时略浪费，但通常能通过。

---

## 解法二：快慢指针 + 交换（推荐，最优）

### 思路

- `slow`：下一个非零元素应放置的位置
- `fast`：扫描整个数组

当 `nums[fast] != 0` 时，交换 `nums[slow]` 与 `nums[fast]`，然后 `slow++`。

### 代码

```python
from typing import List


class Solution:
    def moveZeroes(self, nums: List[int]) -> None:
        slow = 0
        for fast in range(len(nums)):
            if nums[fast] != 0:
                nums[slow], nums[fast] = nums[fast], nums[slow]
                slow += 1
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n)，每个元素最多被访问、交换一次 |
| 空间 | O(1) |

### 讲解

**示例 walkthrough（nums = [0, 1, 0, 3, 12]）：**

```
初始: [0, 1, 0, 3, 12]  slow=0, fast=0
fast=0: 0，跳过
fast=1: 1≠0，交换 [1,0,0,3,12]，slow=1
fast=2: 0，跳过
fast=3: 3≠0，交换 [1,3,0,0,12]，slow=2
fast=4: 12≠0，交换 [1,3,12,0,0]，slow=3
结果: [1, 3, 12, 0, 0]
```

**为什么保持相对顺序？**  
`slow` 只增不减，非零元素按 `fast` 从左到右的发现顺序依次放到 `slow` 位置，顺序不变。

**交换 vs 覆盖：**  
交换法满足「最小化操作次数」的 follow-up（LeetCode 官方扩展），每个非零元素最多移动一次。

## 解法三：冒泡式右移零（不推荐）

### 思路

遇到 0 就不断与右侧元素交换，把 0 慢慢「冒」到右边。

### 代码

```python
from typing import List


class Solution:
    def moveZeroes(self, nums: List[int]) -> None:
        n = len(nums)
        for i in range(n):
            if nums[i] == 0:
                for j in range(i, n - 1):
                    nums[j], nums[j + 1] = nums[j + 1], nums[j]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n²) |
| 空间 | O(1) |

### 讲解

能解决问题但效率差，仅作对比。

---

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 收集 + 填零 | O(n) | O(1) | 易懂 |
| 快慢指针交换 | O(n) | O(1) | 面试首选，操作更少 |
| 冒泡右移 | O(n²) | O(1) | 不推荐 |

---

## 常见错误

1. **破坏相对顺序**：不能把非零元素排序，必须保持原顺序。
2. **返回新数组**：题目要求**原地**修改，返回类型为 `void`。
3. **slow 与 fast 混淆**：应让 `fast` 遍历，`slow` 指向写入位置。
4. **全零数组**：如 `[0,0,0]`，结果仍为 `[0,0,0]`，不要越界。
