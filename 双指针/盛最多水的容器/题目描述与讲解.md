# 盛最多水的容器（Container With Most Water）

- 题目链接：https://leetcode.cn/problems/container-with-most-water/
- 难度：中等
- 标签：数组、双指针、贪心

## 题目描述

给定一个长度为 `n` 的整数数组 `height`。有 `n` 条垂线，第 `i` 条线的两个端点是 `(i, 0)` 和 `(i, height[i])`。

找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。

返回容器可以储存的最大水量。

**说明：** 你不能倾斜容器。

### 示例

```
输入：height = [1,8,6,2,5,4,8,3,7]
输出：49
解释：图中垂直线代表输入数组 [1,8,6,2,5,4,8,3,7]。
在此情况下，容器可以容纳 7 单位的水（垂直线 1 与 8 之间），即 maxArea = 49。

输入：height = [1,1]
输出：1
```

### 约束

- `n == height.length`
- `2 <= n <= 10^5`
- `0 <= height[i] <= 10^4`

---

## 问题建模

选左右两条线 `left` 和 `right`（`left < right`），容器：

- **宽度**：`w = right - left`
- **高度**：`h = min(height[left], height[right])`（水不能溢出，受短板限制）
- **面积**：`area = h × w`

目标：最大化 `area`。

---

## 解法一：暴力枚举（最容易想到）

### 思路

枚举所有 `(i, j)` 对（`i < j`），计算面积，取最大值。

### 代码

```python
from typing import List


class Solution:
    def maxArea(self, height: List[int]) -> int:
        n = len(height)
        ans = 0
        for i in range(n):
            for j in range(i + 1, n):
                h = min(height[i], height[j])
                w = j - i
                ans = max(ans, h * w)
        return ans
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n²) |
| 空间 | O(1) |

### 讲解

暴力法能 AC 小数据，但 `n` 最大 10⁵ 时会超时。  
它的价值在于：先确认「面积公式」写对，再思考如何优化。

---

## 解法二：双指针（推荐，最优）

### 思路

用 `left = 0`、`right = n - 1` 从两端向中间收缩。

每次计算当前 `[left, right]` 的面积后，**移动较短的那一侧指针**：

- 若移动较高的一侧：宽度 `w` 变小，高度 `h` 仍受短板限制（不会变高）→ 面积一定变小或不变，**不可能更优**。
- 若移动较短的一侧：宽度变小，但短板有机会变高 → **仍可能出现更大面积**。

因此每次只移动短板一侧，不会漏掉最优解。

### 代码

```python
from typing import List


class Solution:
    def maxArea(self, height: List[int]) -> int:
        left, right = 0, len(height) - 1
        ans = 0

        while left < right:
            h = min(height[left], height[right])
            w = right - left
            ans = max(ans, h * w)

            if height[left] < height[right]:
                left += 1
            else:
                right -= 1

        return ans
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n)，两个指针最多各移动 n 次 |
| 空间 | O(1) |

### 图解（示例 height = [1,8,6,2,5,4,8,3,7]）

```
索引:  0  1  2  3  4  5  6  7  8
高度:  1  8  6  2  5  4  8  3  7
       ↑                       ↑
     left                   right
面积 = min(1,7) × 8 = 8，left 更短 → left++
```

不断收缩，最终会得到最大面积 **49**（下标 1 与 8，即高度 8 与 7，宽度 7）。

### 正确性直觉

固定 `left` 时，若 `height[left] < height[right]`，那么所有 `(left, k)`（`k < right`）的面积都不会超过以 `height[left]` 为高的当前面积（因为宽度更小）。  
所以可以放心把 `left` 右移，不会丢掉以 `left` 为左边界的最优解。

对称地，当 `height[right]` 更短时，可以安全地 `right--`。

---

## 解法三：为什么「移动长板」是错的？（加深理解）

假设 `height[left] < height[right]`，若移动 `right`：

- 新宽度：`right' - left < right - left`（变窄）
- 新高度：`min(height[left], height[right']) ≤ height[left]`（仍受 left 限制，不会更高）

因此新面积 ≤ `height[left] × (right' - left) < height[left] × (right - left)`，**严格不如当前**。  
移动长板只会浪费一次枚举机会。

| 暴力枚举 | O(n²) | O(1) | 理解题意、验证公式 |
| 双指针 | O(n) | O(1) | 面试与提交首选 |

---

## 常见错误

1. **高度取 max 而不是 min**：水会溢出，必须用两条线中较短的高度。
2. **移动较高的一侧**：会漏掉最优解的逻辑保障，且面积不会变好。
3. **与「接雨水」混淆**：接雨水是求总水量（按列累加），本题是选两条线围成**一个**容器。

## 举一反三

- **接雨水**（Trapping Rain Water）：单调栈或双指针，模型不同
- **三数之和**：排序 + 双指针
- **移动零**：快慢指针，同属指针技巧 family
