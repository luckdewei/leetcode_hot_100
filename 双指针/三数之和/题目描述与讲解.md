# 三数之和（3Sum）

- 题目链接：https://leetcode.cn/problems/3sum/
- 难度：中等
- 标签：数组、双指针、排序

## 题目描述

给你一个整数数组 `nums`，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k`，同时还满足 `nums[i] + nums[j] + nums[k] == 0`。

请你返回所有和为 `0` 且**不重复**的三元组。

**注意：** 答案中不可以包含重复的三元组。

### 示例

```
输入：nums = [-1,0,1,2,-1,-4]
输出：[[-1,-1,2],[-1,0,1]]
解释：
nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0
nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0
nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0
不同的三元组是 [-1,0,1] 和 [-1,-1,2]。
注意，输出的三元组必须按字典序排序。

输入：nums = [0,1,1]
输出：[]

输入：nums = [0,0,0]
输出：[[0,0,0]]
```

### 约束

- `3 <= nums.length <= 3000`
- `-10^5 <= nums[i] <= 10^5`

## 解法一：排序 + 双指针（推荐，最优）

### 思路

1. 将数组排序。
2. 固定第一个数 `nums[i]`，在 `[i+1, n-1]` 上用双指针找两数之和为 `-nums[i]`。
3. 去重：固定项、左指针、右指针在移动时都要跳过重复值。

### 代码

```python
from typing import List


class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        nums.sort()
        n = len(nums)
        ans = []

        for i in range(n - 2):
            if i > 0 and nums[i] == nums[i - 1]:
                continue

            left, right = i + 1, n - 1
            while left < right:
                total = nums[i] + nums[left] + nums[right]
                if total == 0:
                    ans.append([nums[i], nums[left], nums[right]])
                    left += 1
                    right -= 1
                    while left < right and nums[left] == nums[left - 1]:
                        left += 1
                    while left < right and nums[right] == nums[right + 1]:
                        right -= 1
                elif total < 0:
                    left += 1
                else:
                    right -= 1

        return ans
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n²)，外层 n 次，内层双指针 O(n) |
| 空间 | O(1) 额外空间（不计排序和输出） |

### 讲解

这是「两数之和」在排序数组上的自然延伸。

**双指针移动规则：**

- `sum < 0`：和太小，`left++`（需要更大的数）
- `sum > 0`：和太大，`right--`
- `sum == 0`：记录答案，`left++` 且 `right--`，并跳过重复

**去重三处：**

1. `i`：`if i > 0 and nums[i] == nums[i-1]: continue`
2. 找到答案后 `left` 跳过相同
3. 找到答案后 `right` 跳过相同

**示例（nums = [-1, 0, 1, 2, -1, -4]，排序后 [-4, -1, -1, 0, 1, 2]）：**

```
i=0, nums[i]=-4: 需要 sum=4，left/right 找不到 → 无
i=1, nums[i]=-1: 需要 sum=1
  left=2(-1), right=5(2): sum=0 → [-1,-1,2]
  left=3(0), right=4(1): sum=0 → [-1,0,1]
```

## 解法二：暴力枚举 + set 去重

### 思路

三层循环枚举 `i < j < k`，和为 0 则加入 set（元组去重），最后转列表。

### 代码

```python
from typing import List


class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        nums.sort()
        n = len(nums)
        ans_set = set()
        for i in range(n - 2):
            for j in range(i + 1, n - 1):
                for k in range(j + 1, n):
                    if nums[i] + nums[j] + nums[k] == 0:
                        ans_set.add((nums[i], nums[j], nums[k]))
        return [list(t) for t in ans_set]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n³) |
| 空间 | O(n) 用于 set |

### 讲解

n 最大 3000，O(n³) 约 27×10⁹ 次运算，**必定超时**。仅用于理解「去重」需求。

## 解法三：哈希表（两数之和扩展）

### 思路

固定 `i`，对 `j > i` 用哈希表找 `-(nums[i]+nums[j])`，配合排序和相邻去重。

### 代码

```python
from typing import List


class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        nums.sort()
        n = len(nums)
        ans = []
        seen = set()

        for i in range(n - 2):
            if i > 0 and nums[i] == nums[i - 1]:
                continue
            need = -nums[i]
            used = set()
            for j in range(i + 1, n):
                complement = need - nums[j]
                if complement in used:
                    triplet = (nums[i], complement, nums[j])
                    if triplet not in seen:
                        seen.add(triplet)
                        ans.append(list(triplet))
                used.add(nums[j])

        return ans
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n²) |
| 空间 | O(n) |

### 讲解

可行但去重比双指针繁琐，面试一般直接讲排序 + 双指针。

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 排序 + 双指针 | O(n²) | O(1) | 面试首选 |
| 暴力 + set | O(n³) | O(n) | 理解去重 |
| 哈希扩展 | O(n²) | O(n) | 备选 |

## 常见错误

1. **不去重**：`[-1,-1,2]` 和 `[-1,0,1]` 各只应出现一次。
2. **双指针移动方向反了**：和太小应增大 left，不是减小 right。
3. **忘记先排序**：双指针依赖有序性。
4. **返回单个三元组而非列表的列表**：格式为 `List[List[int]]`。
