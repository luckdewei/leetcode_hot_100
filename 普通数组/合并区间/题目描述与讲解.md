# 合并区间（Merge Intervals）

- 题目链接：https://leetcode.cn/problems/merge-intervals/
- 难度：中等
- 标签：数组、排序

## 题目描述

以数组 `intervals` 表示若干区间的集合，其中 `intervals[i] = [start_i, end_i]`。请你合并所有重叠的区间，并返回一个不重叠的区间数组。

### 示例

```
输入：intervals = [[1,3],[2,6],[8,10],[15,18]]
输出：[[1,6],[8,10],[15,18]]

输入：intervals = [[1,4],[4,5]]
输出：[[1,5]]

输入：intervals = [[1,4],[0,4]]
输出：[[0,4]]
```

### 约束

- `1 <= intervals.length <= 10^4`
- `intervals[i].length == 2`
- `0 <= start_i <= end_i <= 10^4`

---

## 解法一：排序后合并（推荐）

### 思路

按起点升序排序。遍历区间，若当前起点 <= 结果中最后一个区间的终点，则合并（更新终点为较大值）；否则追加新区间。

### 代码

```python
from typing import List


class Solution:
    def merge(self, intervals: List[List[int]]) -> List[List[int]]:
        if not intervals:
            return []
        intervals.sort(key=lambda x: x[0])
        merged = [intervals[0]]
        for start, end in intervals[1:]:
            if start <= merged[-1][1]:
                merged[-1][1] = max(merged[-1][1], end)
            else:
                merged.append([start, end])
        return merged
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n log n) |
| 空间 | O(1) 额外（不含输出） |

### 讲解

排序后重叠区间必然相邻。注意 `[1,4],[4,5]` 端点相等也算重叠。

## 解法二：按终点排序

### 思路

按终点排序，维护当前合并区间的起点和终点，类似区间调度。

### 代码

```python
from typing import List


class Solution:
    def merge(self, intervals: List[List[int]]) -> List[List[int]]:
        intervals.sort(key=lambda x: x[1])
        merged = []
        cur_start, cur_end = intervals[0]
        for start, end in intervals[1:]:
            if start <= cur_end:
                cur_end = max(cur_end, end)
            else:
                merged.append([cur_start, cur_end])
                cur_start, cur_end = start, end
        merged.append([cur_start, cur_end])
        return merged
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n log n) |
| 空间 | O(1) |

### 讲解

按终点排序也能合并，但不如按起点直观。面试推荐按起点排序。

## 解法三：暴力合并（理解用）

### 思路

反复找两个可合并区间合并，直到无法合并。

### 代码

```python
from typing import List


class Solution:
    def merge(self, intervals: List[List[int]]) -> List[List[int]]:
        if not intervals:
            return []
        changed = True
        while changed:
            changed = False
            res = []
            used = [False] * len(intervals)
            for i in range(len(intervals)):
                if used[i]:
                    continue
                s, e = intervals[i]
                for j in range(i + 1, len(intervals)):
                    if used[j]:
                        continue
                    s2, e2 = intervals[j]
                    if s2 <= e:
                        s, e = min(s, s2), max(e, e2)
                        used[j] = True
                        changed = True
                res.append([s, e])
            intervals = res
        return sorted(res, key=lambda x: x[0])
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n²) 或更高 |
| 空间 | O(n) |

### 讲解

效率低，仅帮助理解合并条件。

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 按起点排序 | O(n log n) | O(1) | 面试首选 |
| 按终点排序 | O(n log n) | O(1) | 替代写法 |
| 暴力 | O(n²) | O(n) | 理解用 |

---

## 常见错误

1. **合并条件用 `<` 而非 `<=`**：端点相等 `[1,4],[4,5]` 应合并。
2. **忘记更新终点为 max**：`[1,5],[2,3]` 合并后终点是 5 不是 3。
3. **未排序直接合并**：乱序区间会漏合并。
