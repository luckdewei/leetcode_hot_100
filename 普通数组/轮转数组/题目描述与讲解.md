# 轮转数组（Rotate Array）

- 题目链接：https://leetcode.cn/problems/rotate-array/
- 难度：中等
- 标签：数组、数学、双指针

## 题目描述

给定一个整数数组 `nums`，将数组中的元素向右轮转 `k` 个位置，其中 `k` 是非负数。

### 示例

```
输入：nums = [1,2,3,4,5,6,7], k = 3
输出：[5,6,7,1,2,3,4]
解释：向右轮转 1 步 → [7,1,2,3,4,5,6]
     向右轮转 2 步 → [6,7,1,2,3,4,5]
     向右轮转 3 步 → [5,6,7,1,2,3,4]

输入：nums = [-1,-100,3,99], k = 2
输出：[3,99,-1,-100]
```

### 约束

- `1 <= nums.length <= 10^5`
- `-2^31 <= nums[i] <= 2^31 - 1`
- `0 <= k <= 10^5`

**进阶：** 尽可能想出更多解决方案，至少有三种不同的方法。请尝试使用**空间复杂度为 O(1)** 的**原地**算法。

---

## 解法一：额外数组（最容易想到）

### 思路

新位置：`new[(i + k) % n] = nums[i]`。

### 代码

```python
from typing import List


class Solution:
    def rotate(self, nums: List[int], k: int) -> None:
        n = len(nums)
        k %= n
        tmp = [0] * n
        for i in range(n):
            tmp[(i + k) % n] = nums[i]
        nums[:] = tmp
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(n) |

## 解法二：环状替换（原地，较难）

### 思路

从 `i = 0` 开始，将元素放到 `(i + k) % n`，被替换的元素继续放到它的目标位置，形成环。若环长不足 `n`，从下一个未访问下标再开环。需要 O(1) 空间但实现易错。

### 代码

```python
from typing import List


class Solution:
    def rotate(self, nums: List[int], k: int) -> None:
        n = len(nums)
        k %= n
        if k == 0:
            return

        count = 0
        start = 0
        while count < n:
            current = start
            prev = nums[start]
            while True:
                next_idx = (current + k) % n
                nums[next_idx], prev = prev, nums[next_idx]
                current = next_idx
                count += 1
                if current == start:
                    break
            start += 1
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(1) |

## 解法三：三次反转（推荐，最优）

### 思路

右旋 `k` 步等价于：

1. 反转整个数组
2. 反转前 `k` 个元素
3. 反转后 `n - k` 个元素

**例子** `[1,2,3,4,5,6,7]`, k=3：

```
全反转:     [7,6,5,4,3,2,1]
反转前3:    [5,6,7,4,3,2,1]
反转后4:    [5,6,7,1,2,3,4] ✓
```

### 代码

```python
from typing import List


class Solution:
    def rotate(self, nums: List[int], k: int) -> None:
        n = len(nums)
        k %= n
        if k == 0:
            return

        def reverse(left: int, right: int) -> None:
            while left < right:
                nums[left], nums[right] = nums[right], nums[left]
                left += 1
                right -= 1

        reverse(0, n - 1)
        reverse(0, k - 1)
        reverse(k, n - 1)
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n) |
| 空间 | O(1) |

### 正确性

右旋后，原后 `k` 个元素应排在最前 → 全反转后它们落在前 `k` 位但顺序反了 → 再反转前 `k` 位恢复顺序；原前 `n-k` 个元素排在后面，同样需反转恢复。

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 额外数组 | O(n) | O(n) | 最直观 |
| 环状替换 | O(n) | O(1) | 进阶原地 |
| 三次反转 | O(n) | O(1) | 面试首选 |

---

## 常见错误

1. **忘记 `k %= n`**：`k > n` 时旋转等价于 `k % n`。
2. **`k == 0`**：应直接返回，避免无意义操作。
3. **三次反转边界**：第二段是 `[0, k-1]`，第三段是 `[k, n-1]`。

## 举一反三

- **旋转链表**：找新尾、断链、接回
- **字符串左旋**：与本题同一套路
- **轮转数组的最小元素**：可与二分结合（搜索旋转排序数组）
