# 缺失的第一个正数（First Missing Positive）

- 题目链接：https://leetcode.cn/problems/first-missing-positive/
- 难度：困难
- 标签：数组、哈希表

## 题目描述

给你一个未排序的整数数组 `nums`，请你找出其中没有出现的最小的正整数。

请你实现时间复杂度为 **O(n)** 并且只使用**常数级别额外空间**的算法。

### 示例

```
输入：nums = [1,2,0]
输出：3

输入：nums = [3,4,-1,1]
输出：2

输入：nums = [7,8,9,11,12]
输出：1
```

### 约束

- `1 <= nums.length <= 10^5`
- `-2^31 <= nums[i] <= 2^31 - 1`

---

## 关键观察

答案一定在 **`[1, n+1]`** 范围内（`n = len(nums)`）：

- 若 `1..n` 都出现，答案是 `n+1`
- 否则答案是 `1..n` 中第一个缺失的正整数

因此可以把数组当作哈希表：**下标 `i` 对应数字 `i+1` 是否出现**。

---

## 解法一：排序 + 扫描

### 思路

排序后找第一个「值 ≠ 下标+1」的位置。时间 O(n log n)，不满足题目 O(n) 要求。

### 代码

```python
from typing import List


class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        nums.sort()
        expect = 1
        for num in nums:
            if num < expect:
                continue
            if num == expect:
                expect += 1
            else:
                break
        return expect
```

---

## 解法二：布尔数组 / HashSet

### 思路

用 `O(n)` 额外空间标记 `1..n` 哪些出现。正确但空间超标。

### 代码

```python
from typing import List


class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        n = len(nums)
        seen = set(nums)
        for i in range(1, n + 2):
            if i not in seen:
                return i
        return n + 1
```

## 解法三：原地哈希 / 循环置换（推荐，最优）

### 思路

对每个 `nums[i]`，若 `1 <= nums[i] <= n`，则它应该放在下标 `nums[i] - 1` 上。

不断交换直到 `nums[i]` 已在正确位置或值非法：

```python
while 1 <= nums[i] <= n and nums[nums[i]-1] != nums[i]:
    swap(nums[i], nums[nums[i]-1])
```

扫一遍：若 `nums[i] != i+1`，答案为 `i+1`；若全都匹配，答案为 `n+1`。

### 代码

```python
from typing import List


class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        n = len(nums)

        for i in range(n):
            while 1 <= nums[i] <= n and nums[nums[i] - 1] != nums[i]:
                target = nums[i] - 1
                nums[i], nums[target] = nums[target], nums[i]

        for i in range(n):
            if nums[i] != i + 1:
                return i + 1

        return n + 1
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n)，每个元素最多被交换到正确位置一次 |
| 空间 | O(1) |

### 图解（nums = [3,4,-1,1]）

```
目标: 下标0放1, 下标1放2, 下标2放3, 下标3放4

交换后: [1,-1,3,4] → [1,-1,3,4] → [1,2,3,4]（-1 非法跳过）
扫描: i=0→1✓ i=1→2✓ i=2→3✓ i=3→4✓ 全在 → 答案 5? 

等等 [3,4,-1,1]:
i=0: 3 应去下标2 → swap → [-1,4,3,1]
i=0: -1 跳过
i=1: 4 应去下标3 → swap → [-1,1,3,4]
i=1: 1 应去下标0 → swap → [1,-1,3,4]
i=2: 3 已在位
i=3: 4 已在位
扫描: nums[1]=-1≠2 → 答案 2 ✓
```

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 排序 | O(n log n) | O(1) | 不满足题意 |
| HashSet | O(n) | O(n) | 理解映射 |
| 原地置换 | O(n) | O(1) | 面试首选 |

---

## 常见错误

1. **交换条件写错**：必须是 `nums[nums[i]-1] != nums[i]`，避免 `1` 与 `1` 死循环。
2. **答案范围**：别忘了可能是 `n+1`。
3. **负数与超大数**：它们永远不会占据 `1..n` 的位置，可忽略。

---

## 举一反三

- **寻找重复数**：数组作链表，Floyd 判环（值域也是 1..n）
- **数组中重复的数字（剑指）**：原地置换同一套路
- **缺失的数字**：异或或求和，变种题
