# 最小栈（Min Stack）

- 题目链接：https://leetcode.cn/problems/min-stack/
- 难度：中等
- 标签：栈、设计

## 题目描述

设计一个支持 `push`、`pop`、`top` 和 `getMin` 操作的栈，且各操作时间复杂度为 **O(1)**。

实现 `MinStack` 类：
- `MinStack()` 初始化栈
- `void push(int val)` 入栈
- `void pop()` 出栈
- `int top()` 获取栈顶
- `int getMin()` 获取栈中最小值

### 示例

```
输入：["MinStack","push","push","push","getMin","pop","top","getMin"]
     [[],[-2],[0],[-3],[],[],[],[]]
输出：[null,null,null,null,-3,null,0,-2]
```

### 约束

- `-2^31 <= val <= 2^31 - 1`
- `pop`、`top`、`getMin` 在栈非空时调用
- 最多 `3 * 10^4` 次调用

---

## 解法一：值与当前最小值配对存储（推荐）

### 思路

栈中每个元素存 `(val, current_min)`。入栈时 `current_min = min(val, 栈顶current_min)`，出栈时最小值随元素一起弹出。

### 代码

```python
class MinStack:
    def __init__(self):
        self.stack = []  # (value, current_min)

    def push(self, val: int) -> None:
        if not self.stack:
            self.stack.append((val, val))
        else:
            self.stack.append((val, min(val, self.stack[-1][1])))

    def pop(self) -> None:
        self.stack.pop()

    def top(self) -> int:
        return self.stack[-1][0]

    def getMin(self) -> int:
        return self.stack[-1][1]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 各操作 | O(1) |
| 空间 | O(n) |

### 讲解

实现简洁，每个操作 O(1)。空间 O(n) 存冗余最小值，是常见权衡。

## 解法二：辅助最小值栈

### 思路

主栈存值，辅助栈同步存当前最小值。入栈时若 `val <= 辅助栈顶` 则压入辅助栈；出栈时若弹出值等于辅助栈顶则同步弹出。

### 代码

```python
class MinStack:
    def __init__(self):
        self.stack = []
        self.min_stack = []

    def push(self, val: int) -> None:
        self.stack.append(val)
        if not self.min_stack or val <= self.min_stack[-1]:
            self.min_stack.append(val)

    def pop(self) -> None:
        val = self.stack.pop()
        if val == self.min_stack[-1]:
            self.min_stack.pop()

    def top(self) -> int:
        return self.stack[-1]

    def getMin(self) -> int:
        return self.min_stack[-1]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 各操作 | O(1) |
| 空间 | O(n) 最坏 |

### 讲解

辅助栈只在「新的最小值或相等最小值」入栈，重复最小值需用 `<=` 处理连续相同最小值出栈。

## 解法三：差值编码（进阶）

### 思路

栈存与当前最小值的差值，用 long 避免溢出。适用于对空间极度敏感的场景。

### 代码

```python
class MinStack:
    def __init__(self):
        self.stack = []
        self.min_val = None

    def push(self, val: int) -> None:
        if not self.stack:
            self.stack.append(0)
            self.min_val = val
        else:
            self.stack.append(val - self.min_val)
            if val < self.min_val:
                self.min_val = val

    def pop(self) -> None:
        diff = self.stack.pop()
        if diff < 0:
            self.min_val = self.min_val - diff

    def top(self) -> int:
        diff = self.stack[-1]
        return self.min_val if diff < 0 else self.min_val + diff

    def getMin(self) -> int:
        return self.min_val
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 各操作 | O(1) |
| 空间 | O(n) |

### 讲解

技巧性强，面试中解法一、二足够。差值法每个节点只存一个数，但实现易错。

## 解法对比

| 解法 | 空间 | 实现难度 | 适用场景 |
|------|------|----------|----------|
| 配对存储 | O(n) | 低 | 面试首选 |
| 辅助栈 | O(n) | 低 | 同样推荐 |
| 差值编码 | O(n) | 高 | 空间优化讨论 |

---

## 常见错误

1. **辅助栈用 `<` 而非 `<=`**：连续相同最小值出栈时会错。
2. **getMin 单独遍历栈**：违反 O(1) 要求。
3. **pop 后忘记更新最小值**：配对存储法无此问题，单栈法会错。
