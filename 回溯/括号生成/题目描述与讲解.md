# 括号生成（Generate Parentheses）

- 题目链接：https://leetcode.cn/problems/generate-parentheses/
- 难度：中等
- 标签：字符串、回溯、动态规划

## 题目描述

数字 `n` 代表生成括号的对数，请你生成所有可能并且**有效的括号组合**。

### 示例

```
输入：n = 3
输出：["((()))","(()())","(())()","()(())","()()()"]

输入：n = 1
输出：["()"]

输入：n = 2
输出：["(())","()()"]
```

### 约束

- `1 <= n <= 8`

---

## 解法一：回溯 + 左右括号计数（推荐）

### 思路

维护已放左括号数 `open` 和右括号数 `close`。规则：
- 若 `open < n`，可放 `(`
- 若 `close < open`，可放 `)`（保证任意前缀左括号不少于右括号）
- 长度达 `2n` 时收集答案

### 代码

```python
from typing import List


class Solution:
    def generateParenthesis(self, n: int) -> List[str]:
        res, path = [], []

        def backtrack(open_cnt: int, close_cnt: int) -> None:
            if len(path) == 2 * n:
                res.append("".join(path))
                return
            if open_cnt < n:
                path.append("(")
                backtrack(open_cnt + 1, close_cnt)
                path.pop()
            if close_cnt < open_cnt:
                path.append(")")
                backtrack(open_cnt, close_cnt + 1)
                path.pop()

        backtrack(0, 0)
        return res
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(4^n / √n)，第 n 个卡特兰数量级 |
| 空间 | O(n)，递归深度 |

### 讲解

通过 `close < open` 剪枝，避免生成无效串后再过滤。这是括号类回溯的核心约束。

## 解法二：回溯 + 合法性预判

### 思路

不分别计数，每次尝试放 `(` 或 `)`，用辅助函数检查当前 path 是否仍可能扩展为合法串（左括号数 ≥ 右括号数且左括号 ≤ n）。

### 代码

```python
from typing import List


class Solution:
    def generateParenthesis(self, n: int) -> List[str]:
        res = []

        def valid(prefix: str) -> bool:
            bal = 0
            for ch in prefix:
                bal += 1 if ch == "(" else -1
                if bal < 0:
                    return False
            return bal <= n and prefix.count("(") <= n

        def backtrack(path: str) -> None:
            if len(path) == 2 * n:
                if path.count("(") == n:
                    res.append(path)
                return
            for ch in ("(", ")"):
                if valid(path + ch):
                    backtrack(path + ch)

        backtrack("")
        return res
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | 高于解法一，重复校验 |
| 空间 | O(n) |

### 讲解

逻辑正确但效率低，适合理解「有效括号 = 任意前缀 balance ≥ 0」。面试推荐计数法。

## 解法三：动态规划（按长度递推）

### 思路

`dp[i]` 表示 i 对括号的所有合法组合。在 `()` 内包裹 `dp[j]` 的子结构，再拼接 `dp[i-1-j]`。

### 代码

```python
from typing import List


class Solution:
    def generateParenthesis(self, n: int) -> List[str]:
        dp: List[List[str]] = [[] for _ in range(n + 1)]
        dp[0] = [""]
        for i in range(1, n + 1):
            for j in range(i):
                for left in dp[j]:
                    for right in dp[i - 1 - j]:
                        dp[i].append(f"({left}){right}")
        return dp[n]
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(4^n / √n) |
| 空间 | O(4^n / √n) |

### 讲解

卡特兰数的 DP 构造：`(` + 中间 j 对 + `)` + 剩余 i-1-j 对。非回溯思路，拓展对括号 DP 的理解。

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 计数回溯 | O(4^n/√n) | O(n) | 面试首选 |
| 合法性预判 | 较慢 | O(n) | 理解有效前缀 |
| DP 递推 | O(4^n/√n) | O(输出) | 卡特兰数构造 |

## 常见错误

1. **右括号条件写错**：应是 `close < open`，不是 `close < n`。
2. **左括号无上限**：必须 `open < n`，否则左括号过多。
3. **先放右括号再放左**：不剪枝会产生大量无效串。
4. **与「最长有效括号」混淆**：32 题是求最长合法子串长度，非生成全部。
