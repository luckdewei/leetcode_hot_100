# 子集（Subsets）

- 题目链接：https://leetcode.cn/problems/subsets/
- 难度：中等
- 标签：位运算、数组、回溯

## 题目描述

给你一个**整数数组** `nums`，数组中的元素**互不相同**。返回该数组所有可能的**子集**（幂集）。解集**不能**包含重复子集。可以按任意顺序返回。

### 示例

```
输入：nums = [1, 2, 3]
输出：[[],[1],[2],[1,2],[3],[1,3],[2,3],[1,2,3]]

输入：nums = [0]
输出：[[],[0]]
```

### 约束

- `1 <= nums.length <= 10`
- `-10 <= nums[i] <= 10`
- `nums` 中的所有元素**互不相同**

---

## 解法一：回溯枚举（推荐）

### 思路

子集问题用 `start` 控制「当前可选起点」，避免 `[2,1]` 与 `[1,2]` 重复。每进入一层递归，先把当前 `path` 加入结果（包括空集），再枚举 `i ∈ [start, n)` 选或不选下一元素。

### 代码

```python
from typing import List


class Solution:
    def subsets(self, nums: List[int]) -> List[List[int]]:
        res, path = [], []

        def backtrack(start: int) -> None:
            res.append(path[:])
            for i in range(start, len(nums)):
                path.append(nums[i])
                backtrack(i + 1)
                path.pop()

        backtrack(0)
        return res
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n × 2^n)，共 2^n 个子集 |
| 空间 | O(n)，递归栈 + path |

### 讲解

与排列不同，子集**不关心顺序**，只关心「选哪些」。`start = i + 1` 保证只向后选，天然去重。这是组合/子集回溯的统一模板。

## 解法二：二进制枚举

### 思路

用 0 到 2^n - 1 的二进制掩码表示每个元素选/不选。第 `j` 位为 1 则取 `nums[j]`。

### 代码

```python
from typing import List


class Solution:
    def subsets(self, nums: List[int]) -> List[List[int]]:
        n = len(nums)
        res = []
        for mask in range(1 << n):
            subset = [nums[j] for j in range(n) if mask >> j & 1]
            res.append(subset)
        return res
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n × 2^n) |
| 空间 | O(1) 额外（不含输出） |

### 讲解

代码极简，n ≤ 10 时完全可行。理解「每个子集对应唯一 bitmask」有助于掌握幂集本质。

## 解法三：迭代构建

### 思路

初始只有空集 `[[]]`。遍历每个 `num`，把现有每个子集末尾追加 `num` 生成新子集，合并进结果。

### 代码

```python
from typing import List


class Solution:
    def subsets(self, nums: List[int]) -> List[List[int]]:
        res = [[]]
        for num in nums:
            res += [subset + [num] for subset in res]
        return res
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n × 2^n) |
| 空间 | O(n × 2^n) |

### 讲解

每加入一个元素，子集数量翻倍。非递归写法，适合不喜欢回溯的场景。

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 回溯 | O(n × 2^n) | O(n) | 面试首选，可扩展组合问题 |
| 二进制 | O(n × 2^n) | O(1) | n 小、代码短 |
| 迭代 | O(n × 2^n) | O(n × 2^n) | 理解子集倍增 |

## 常见错误

1. **start 写成 0**：会导致 `[1,2]` 和 `[2,1]` 都出现。
2. **只在叶子收集**：子集问题每层都要 `res.append(path[:])`，不是等到选满才收集。
3. **与「子集 II」混淆**：90 题有重复元素，需排序 + 同层去重。
4. **path 未拷贝**：同上，必须用 `path[:]`。
