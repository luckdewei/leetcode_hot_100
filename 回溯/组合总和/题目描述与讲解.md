# 组合总和（Combination Sum）

- 题目链接：https://leetcode.cn/problems/combination-sum/
- 难度：中等
- 标签：数组、回溯

## 题目描述

给你一个**无重复元素**的整数数组 `candidates` 和一个目标整数 `target`，找出 `candidates` 中所有和为 `target` 的组合。

`candidates` 中的**同一个数字可以无限制重复选取**。解集不能包含重复组合。

### 示例

```
输入：candidates = [2, 3, 6, 7], target = 7
输出：[[2,2,3],[7]]

输入：candidates = [2, 3, 5], target = 8
输出：[[2,2,2,2],[2,3,3],[3,5]]

输入：candidates = [2], target = 1
输出：[]
```

### 约束

- `1 <= candidates.length <= 30`
- `2 <= candidates[i] <= 40`
- `candidates` 元素互不相同
- `1 <= target <= 40`

---

## 解法一：回溯 + 排序剪枝（推荐）

### 思路

与子集类似用 `start` 避免 `[2,3]` 和 `[3,2]` 重复。因为可重复选，递归时仍从 `i` 而非 `i+1` 开始。先排序，`x > remain` 时 `break` 剪枝。

### 代码

```python
from typing import List


class Solution:
    def combinationSum(self, candidates: List[int], target: int) -> List[List[int]]:
        candidates.sort()
        res, path = [], []

        def backtrack(start: int, remain: int) -> None:
            if remain == 0:
                res.append(path[:])
                return
            for i in range(start, len(candidates)):
                x = candidates[i]
                if x > remain:
                    break
                path.append(x)
                backtrack(i, remain - x)
                path.pop()

        backtrack(0, target)
        return res
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | O(n^(T/min))，T 为 target，min 为最小候选 |
| 空间 | O(T/min)，递归深度 |

### 讲解

`backtrack(i, ...)` 允许再次选 `candidates[i]`，实现无限次使用。排序后一旦当前数大于剩余和，后续更大，直接 break。

---

## 解法二：回溯不排序（理解用）

### 思路

不排序，仅用 `remain < 0` 作为终止剪枝。逻辑正确但效率较低。

### 代码

```python
from typing import List


class Solution:
    def combinationSum(self, candidates: List[int], target: int) -> List[List[int]]:
        res, path = [], []

        def backtrack(start: int, remain: int) -> None:
            if remain == 0:
                res.append(path[:])
                return
            if remain < 0:
                return
            for i in range(start, len(candidates)):
                path.append(candidates[i])
                backtrack(i, remain - candidates[i])
                path.pop()

        backtrack(0, target)
        return res
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | 高于排序版，分支更多 |
| 空间 | O(T/min) |

### 讲解

帮助理解剪枝的价值：无排序时无法提前 break，会探索更多无效分支。

## 解法三：动态规划（计数/方案数）

### 思路

若只求**组合个数**（非列举方案），可用 DP：`dp[s]` 表示和为 s 的组合数，遍历每个 candidate 更新。

### 代码

```python
from typing import List


class Solution:
    def combinationSum(self, candidates: List[int], target: int) -> List[List[int]]:
        # 本题要求列举方案，DP 列举需额外记录路径，此处展示计数思路
        dp = [0] * (target + 1)
        dp[0] = 1
        for x in candidates:
            for s in range(x, target + 1):
                dp[s] += dp[s - x]
        # 计数结果：dp[target]；列举方案仍推荐回溯
        return [] if dp[target] == 0 else self._backtrack_list(candidates, target)

    def _backtrack_list(self, candidates, target):
        candidates.sort()
        res, path = [], []
        def bt(start, remain):
            if remain == 0:
                res.append(path[:]); return
            for i in range(start, len(candidates)):
                if candidates[i] > remain: break
                path.append(candidates[i])
                bt(i, remain - candidates[i])
                path.pop()
        bt(0, target)
        return res
```

### 复杂度

| 指标 | 复杂度 |
|------|--------|
| 时间 | DP 计数 O(n × target)；列举仍 O(n^(T/min)) |
| 空间 | O(target) |

### 讲解

完全背包计数模型。本题要输出所有组合，DP 列举较繁琐，回溯更自然；DP 适合「518 零钱兑换 II」类计数题。

---

## 解法对比

| 解法 | 时间 | 空间 | 适用场景 |
|------|------|------|----------|
| 排序回溯 | O(n^(T/min)) | O(T/min) | 面试首选 |
| 无排序回溯 | 更慢 | O(T/min) | 理解剪枝 |
| DP 计数 | O(n × target) | O(target) | 只求个数时 |

---

## 常见错误

1. **递归传 `i+1`**：会变成每个数最多用一次，变成「组合总和 II」逻辑。
2. **未排序就 break**：`break` 依赖升序，否则剪枝错误。
3. **`remain == 0` 后继续**：应在收集后立即 return（for 循环自然结束即可）。
4. **与「组合总和 II」混淆**：40 题不可重复选同一元素，且需去重。
